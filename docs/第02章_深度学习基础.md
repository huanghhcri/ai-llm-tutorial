## 二、核心知识点

### 2.1 从生物神经元到人工神经元

#### 类比

养老院里，护士每天要判断"这位长者是否需要紧急就医"。她会综合考虑多个因素：

- 心率是否异常？（权重最大）
- 体温是否过高？
- 血氧是否过低？
- 长者是否有意识？

护士的大脑会给每个因素分配不同的"重要程度"（权重），加权求和后做出判断。**人工神经元做的就是同样的事情。**

#### 数学定义

一个人工神经元的计算过程：

```
输入: x₁, x₂, ..., xₙ（特征值）
权重: w₁, w₂, ..., wₙ（每个特征的重要程度）
偏置: b（阈值调整项）

加权求和: z = w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + b
激活输出: a = f(z)（f 是激活函数）
```

用养老院的例子：

```
判断是否需要紧急就医：

输入：心率=120, 体温=38.5, 血氧=88, 意识=1(清醒)
权重：w₁=0.4, w₂=0.25, w₃=0.3, w₄=0.05
偏置：b = -30

z = 0.4×120 + 0.25×38.5 + 0.3×88 + 0.05×1 + (-30)
  = 48 + 9.625 + 26.4 + 0.05 - 30
  = 54.075

如果激活函数是"阈值函数"：z > 50 → 需要就医 ✓
```

#### 代码实现

```python
import numpy as np

# 手动实现一个神经元（对标 C# 你会写一个方法）
def neuron(inputs: list[float], weights: list[float], bias: float) -> float:
    """
    单个神经元的计算。
    
    Args:
        inputs: 输入特征 [心率, 体温, 血氧, 意识状态]
        weights: 权重 [心率权重, 体温权重, 血氧权重, 意识权重]
        bias: 偏置项
    
    Returns:
        加权求和结果
    """
    # C# 等价写法：inputs.Zip(weights, (x, w) => x * w).Sum() + bias
    z = sum(x * w for x, w in zip(inputs, weights)) + bias
    return z

# 养老院场景：判断是否需要紧急就医
vital_signs = [120, 38.5, 88, 1]      # 心率, 体温, 血氧, 意识(1=清醒)
weights = [0.4, 0.25, 0.3, 0.05]
bias = -30

result = neuron(vital_signs, weights, bias)
print(f"加权求和结果: {result}")         # 54.075
print(f"是否需要就医: {result > 50}")    # True
```

### 2.2 激活函数 — 为什么要"非线性"

#### 类比

如果没有激活函数，神经网络无论多少层都只是一个线性函数（矩阵乘法的叠加）。这就像养老院的护理方案——如果所有长者都用同一套线性规则（"年龄越大护理等级越高"），就无法处理复杂情况（比如一位 95 岁但身体硬朗的长者，和一位 70 岁但有多重慢病的长者，他们的护理需求完全不同）。激活函数引入非线性，让模型能学习复杂的决策边界。

#### 常用激活函数

```python
import numpy as np
import matplotlib.pyplot as plt

def plot_activations():
    """可视化常见激活函数"""
    x = np.linspace(-5, 5, 200)
    
    # 1. Sigmoid — 将输出压缩到 (0, 1)
    # 类比：把任何数值转化为"概率"
    # 公式：σ(x) = 1 / (1 + e^(-x))
    sigmoid = 1 / (1 + np.exp(-x))
    
    # 2. Tanh — 将输出压缩到 (-1, 1)
    # 类比：把数值标准化到"正负对称"范围
    # 公式：tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))
    tanh = np.tanh(x)
    
    # 3. ReLU — 负数变 0，正数不变（最常用！）
    # 类比：只关注"有异常"的指标，正常的忽略
    # 公式：ReLU(x) = max(0, x)
    relu = np.maximum(0, x)
    
    # 4. Leaky ReLU — 负数给一个很小的斜率
    # 类比：异常指标也保留一点信息，不要完全丢弃
    # 公式：LeakyReLU(x) = x if x > 0 else 0.01x
    leaky_relu = np.where(x > 0, x, 0.01 * x)
    
    return x, sigmoid, tanh, relu, leaky_relu
```

#### 激活函数对比

| 激活函数 | 公式 | 输出范围 | 优点 | 缺点 | 典型用途 |
|---------|------|---------|------|------|---------|
| **Sigmoid** | 1/(1+e^(-x)) | (0, 1) | 输出可解释为概率 | 梯度消失、非零中心 | 二分类输出层 |
| **Tanh** | (e^x-e^(-x))/(e^x+e^(-x)) | (-1, 1) | 零中心 | 梯度消失 | 隐藏层（RNN） |
| **ReLU** | max(0, x) | [0, +∞) | 计算快、缓解梯度消失 | 死亡神经元 | **隐藏层首选** |
| **Leaky ReLU** | max(0.01x, x) | (-∞, +∞) | 解决死亡神经元 | 多一个超参数 | ReLU 的替代 |
| **GELU** | x·Φ(x) | (-∞, +∞) | 平滑、Transformer 常用 | 计算稍慢 | **Transformer 隐藏层** |
| **SiLU/Swish** | x·σ(x) | (-∞, +∞) | 平滑、自门控 | 计算稍慢 | LLaMA 系列 |

> **面试高频考点**：为什么 ReLU 是默认选择？因为 Sigmoid/Tanh 在输入值很大或很小时梯度接近 0（梯度消失），导致深层网络无法训练。ReLU 在正区间梯度恒为 1，完美解决了这个问题。

### 2.3 神经网络结构

#### 类比

养老院的"健康评估流水线"就是一个神经网络：

```
第一层（输入层）：收集原始数据
  → 心率、体温、血压、血糖、血氧...

第二层（隐藏层 1）：初步特征提取
  → "心率异常"、"血压偏高"、"血糖异常"...

第三层（隐藏层 2）：综合判断
  → "心血管风险"、"代谢风险"、"整体健康评分"...

第四层（输出层）：最终决策
  → 护理等级：自理 / 半护理 / 全护理 / 特护
```

每一层的神经元与下一层的神经元**全连接**（每个输出都参与下一层的每个输入），这就是"全连接层"（Dense Layer / Linear Layer）。

#### 代码：用 PyTorch 搭建第一个神经网络

```python
import torch
import torch.nn as nn


class HealthRiskNet(nn.Module):
    """
    养老院健康风险评估神经网络。
    
    结构：
        输入层(6个特征) → 隐藏层1(32个神经元) → 隐藏层2(16个神经元) → 输出层(4个类别)
    
    输入特征：心率、收缩压、舒张压、体温、血糖、血氧
    输出类别：低风险(0)、中风险(1)、高风险(2)、极高风险(3)
    """
    
    def __init__(self):
        super().__init__()   # 对标 C# 的 base()
        
        # 定义网络层（对标 C# 的构造函数中初始化各层）
        self.network = nn.Sequential(
            # 第一层：输入(6) → 隐藏层(32)
            # nn.Linear ≈ C# 的全连接层，参数为 (输入维度, 输出维度)
            nn.Linear(6, 32),
            nn.ReLU(),        # 激活函数
            nn.Dropout(0.2),  # 随机丢弃 20% 神经元，防止过拟合
            
            # 第二层：隐藏层(32) → 隐藏层(16)
            nn.Linear(32, 16),
            nn.ReLU(),
            nn.Dropout(0.2),
            
            # 输出层：隐藏层(16) → 输出(4)
            nn.Linear(16, 4),
            # 注意：这里不加 Softmax，因为 CrossEntropyLoss 内部会处理
        )
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        前向传播（数据从输入到输出的计算过程）。
        对标 C# 接口中的方法定义，PyTorch 会自动调用。
        
        Args:
            x: 输入张量，形状为 (batch_size, 6)
        Returns:
            输出张量，形状为 (batch_size, 4)
        """
        return self.network(x)


# 创建模型实例
model = HealthRiskNet()

# 打印模型结构（对标 C# 查看类的属性和方法）
print(model)

# 统计参数量
total_params = sum(p.numel() for p in model.parameters())
print(f"\n模型总参数量: {total_params:,}")
```

输出：
```
HealthRiskNet(
  (network): Sequential(
    (0): Linear(in_features=6, out_features=32, bias=True)
    (1): ReLU()
    (2): Dropout(p=0.2, inplace=False)
    (3): Linear(in_features=32, out_features=16, bias=True)
    (4): ReLU()
    (5): Dropout(p=0.2, inplace=False)
    (6): Linear(in_features=16, out_features=4, bias=True)
  )
)

模型总参数量: 820
```

> **参数量计算**：第一层 Linear(6, 32) 有 6×32 + 32 = 224 个参数（权重矩阵 + 偏置向量），第二层 32×16 + 16 = 528，第三层 16×4 + 4 = 68，总共 224 + 528 + 68 = **820**。Dropout 和 ReLU 没有可学习参数。

### 2.4 损失函数 — "预测有多差"

#### 类比

养老院每月考核护理员的工作质量。如何量化"护理质量有多差"？

- **分类问题**（判断护理等级对不对）→ 数错了几个？→ **交叉熵损失**
- **回归问题**（预测血压值偏差多少）→ 预测值与真实值差多少？→ **均方误差**

损失函数就是模型的"考试成绩"——损失越小，模型预测越准。

#### 常用损失函数

```python
import torch
import torch.nn as nn

# ==========================================
# 1. 均方误差（MSE）— 回归问题
# ==========================================
# 场景：预测长者的血压值
mse_loss = nn.MSELoss()

predicted_bp = torch.tensor([125.0, 140.0, 110.0])   # 预测的收缩压
actual_bp = torch.tensor([120.0, 155.0, 115.0])       # 实际的收缩压

loss = mse_loss(predicted_bp, actual_bp)
print(f"MSE 损失: {loss.item():.2f}")
# MSE = mean((125-120)² + (140-155)² + (110-115)²) = mean(25+225+25) = 91.67

# C# 等价写法：
# var mse = predictions.Zip(actuals, (p, a) => Math.Pow(p - a, 2)).Average();

# ==========================================
# 2. 交叉熵损失（CrossEntropyLoss）— 分类问题
# ==========================================
# 场景：判断护理等级（0=自理, 1=半护理, 2=全护理, 3=特护）
cross_entropy = nn.CrossEntropyLoss()

# 模型输出（未归一化的 logits，4个类别的得分）
logits = torch.tensor([
    [2.1, 0.5, -1.2, -3.0],   # 样本1：模型认为是"自理"
    [0.3, 1.8, 0.9, -0.5],    # 样本2：模型认为是"半护理"
])

# 真实标签（类别的索引）
labels = torch.tensor([0, 1])  # 样本1确实是"自理"，样本2确实是"半护理"

loss = cross_entropy(logits, labels)
print(f"交叉熵损失: {loss.item():.4f}")

# ==========================================
# 3. 二元交叉熵（BCELoss）— 二分类问题
# ==========================================
# 场景：判断是否需要紧急就医（0=不需要, 1=需要）
bce_loss = nn.BCEWithLogitsLoss()  # 内置 Sigmoid，更数值稳定

binary_logits = torch.tensor([1.5, -0.8, 3.2])      # 原始输出
binary_labels = torch.tensor([1.0, 0.0, 1.0])        # 真实标签

loss = bce_loss(binary_logits, binary_labels)
print(f"二元交叉熵损失: {loss.item():.4f}")
```

#### 损失函数选择指南

| 场景 | 损失函数 | 养老院示例 |
|------|---------|-----------|
| 回归（预测连续值） | `nn.MSELoss()` | 预测血压、血糖、BMI |
| 二分类 | `nn.BCEWithLogitsLoss()` | 是否需要就医、是否跌倒 |
| 多分类 | `nn.CrossEntropyLoss()` | 护理等级(4类)、健康风险等级(4类) |
| 多标签分类 | `nn.BCEWithLogitsLoss()` | 同时判断多种慢病(有/无高血压+有/无糖尿病+...) |

### 2.5 反向传播 — 模型如何"学习"

#### 类比

养老院的护理员小王第一次给长者做健康评估，给出了"自理"的判断，但医生复核后说是"半护理"（**损失**）。小王会反思："我哪里判断错了？心率指标我可能权重给低了，血压指标权重给高了。"然后调整自己的判断标准（**更新权重**）。这个"反思→调整"的过程就是反向传播。

#### 数学原理（简化版）

```
前向传播：输入 → 各层计算 → 预测值 → 计算损失
反向传播：损失 → 链式法则求梯度 → 各层的权重应该往哪个方向调

核心公式（梯度下降）：
    w_new = w_old - learning_rate × ∂Loss/∂w

    learning_rate（学习率）：每次调整的步长
        太大 → 跳过最优解（震荡）
        太小 → 收敛太慢
        典型值：0.001 ~ 0.01
```

#### 代码演示

```python
import torch
import torch.nn as nn

# 创建一个简单的模型
model = nn.Linear(2, 1)  # 2个输入，1个输出

# 模拟数据：长者的 [年龄, 慢病数量] → 风险评分
x = torch.tensor([[78.0, 2.0], [85.0, 4.0], [65.0, 1.0]])
y_true = torch.tensor([[60.0], [85.0], [30.0]])

# 定义损失函数和优化器
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.0001)  # 随机梯度下降

# 训练循环（这就是深度学习的核心！）
for epoch in range(100):
    # ======== 前向传播 ========
    y_pred = model(x)            # 计算预测值
    loss = criterion(y_pred, y_true)  # 计算损失
    
    # ======== 反向传播 ========
    optimizer.zero_grad()        # 清零梯度（对标 C#：每轮开始前重置状态）
    loss.backward()              # 计算梯度（反向传播的核心！）
    optimizer.step()             # 更新权重
    
    if epoch % 20 == 0:
        print(f"第 {epoch} 轮, 损失: {loss.item():.4f}")

# 训练完成后，用模型预测
with torch.no_grad():           # 预测时不需要计算梯度
    new_member = torch.tensor([[90.0, 5.0]])  # 90岁，5种慢病
    risk_score = model(new_member)
    print(f"\n预测风险评分: {risk_score.item():.1f}")
```

输出示例：
```
第 0 轮, 损失: 4567.3214
第 20 轮, 损失: 125.8932
第 40 轮, 损失: 45.2103
第 60 轮, 损失: 12.5647
第 80 轮, 损失: 3.8921

预测风险评分: 92.3
```

#### 训练流程图

```
┌─────────────────────────────────────────────────────┐
│                    训练循环（每个 Epoch）               │
│                                                       │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐       │
│  │ 前向传播  │───→│ 计算损失  │───→│ 反向传播  │       │
│  │ Forward  │    │ Loss     │    │ Backward │       │
│  └──────────┘    └──────────┘    └──────────┘       │
│       ↑                                   │          │
│       │           ┌──────────┐            │          │
│       └───────────│ 更新权重  │←───────────┘          │
│                   │ Optimizer│                       │
│                   └──────────┘                       │
│                                                       │
│  重复 N 个 Epoch，直到损失不再下降                      │
└─────────────────────────────────────────────────────┘
```

### 2.6 优化器 — 用什么策略调参数

#### 类比

优化器就像护理培训的"改进策略"：

- **SGD**（随机梯度下降）：每次只看一个样本调整——简单但慢
- **SGD + 动量**：像小球滚下山坡，有惯性——不容易卡在局部最优
- **Adam**：自适应学习率——每个参数自动调整步长（**默认首选**）

#### 常用优化器

```python
import torch.optim as optim

model = HealthRiskNet()  # 之前定义的模型

# 1. SGD — 最基础的优化器
optimizer_sgd = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
# momentum=0.9：动量因子，帮助跳出局部最优

# 2. Adam — 最常用的优化器（推荐默认使用）
optimizer_adam = optim.Adam(model.parameters(), lr=0.001)
# 自适应学习率：每个参数有不同的学习率

# 3. AdamW — Adam + 权重衰减（Transformer 训练首选）
optimizer_adamw = optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)
# weight_decay：正则化，防止权重过大

# 4. 学习率调度器 — 训练过程中动态调整学习率
scheduler = optim.lr_scheduler.StepLR(optimizer_adam, step_size=10, gamma=0.5)
# 每 10 个 epoch，学习率乘以 0.5
```

#### 优化器选择指南

| 优化器 | 特点 | 推荐场景 |
|--------|------|---------|
| SGD + Momentum | 简单、泛化好 | CV 领域经典选择 |
| **Adam** | 自适应、收敛快 | **通用默认选择** |
| **AdamW** | Adam + 正则化 | **Transformer/大模型训练首选** |

> **面试考点**：Adam 和 AdamW 的区别——Adam 的权重衰减实现有缺陷（L2 正则化与自适应学习率耦合），AdamW 将权重衰减从梯度更新中解耦，效果更好。几乎所有大模型训练都用 AdamW。

### 2.7 过拟合与正则化

#### 类比

养老院新来了一个护理员小李，培训时把所有长者的特殊情况都背下来了（过拟合）。结果换个养老院就不会护理了——他记住了"张大爷怕冷要多盖被子"，但没学会"体弱的长者要注意保暖"这个通用规律。

过拟合 = 模型在训练数据上表现很好，但在新数据上表现差。

#### 解决方案

```python
import torch.nn as nn

class RegularizedNet(nn.Module):
    """带正则化的健康风险评估网络"""
    
    def __init__(self):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(6, 64),
            nn.ReLU(),
            nn.Dropout(0.3),      # 方案1：Dropout — 随机丢弃 30% 神经元
            nn.BatchNorm1d(64),   # 方案2：批归一化 — 稳定训练过程
            
            nn.Linear(64, 32),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.BatchNorm1d(32),
            
            nn.Linear(32, 4),
        )
    
    def forward(self, x):
        return self.network(x)


# 方案3：早停法（Early Stopping）— 验证集损失不再下降时停止训练
class EarlyStopping:
    """早停机制"""
    
    def __init__(self, patience: int = 5, min_delta: float = 0.001):
        """
        Args:
            patience: 容忍多少个 epoch 没有改善
            min_delta: 最小改善幅度
        """
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = float('inf')
        self.should_stop = False
    
    def check(self, val_loss: float) -> bool:
        """检查是否应该停止训练"""
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
        else:
            self.counter += 1
            if self.counter >= self.patience:
                self.should_stop = True
        return self.should_stop
```

#### 正则化方法总结

| 方法 | 原理 | 类比 |
|------|------|------|
| **Dropout** | 训练时随机关闭部分神经元 | 每次考核随机换一批护理员，避免依赖特定个人 |
| **BatchNorm** | 归一化每层的输入 | 把所有指标标准化到同一量级（心率 60-100 vs 血糖 3-11） |
| **L2 正则化** | 惩罚过大的权重 | 限制护理员的"极端判断"，鼓励温和的决策 |
| **早停法** | 验证集不再提升时停止 | 培训效果不提升时及时停止，别过度培训 |
| **数据增强** | 增加训练数据多样性 | 让护理员接触更多不同类型的长者案例 |

## 四、知识点总结

| 概念 | 一句话解释 | 关键要点 |
|------|-----------|---------|
| **人工神经元** | 对输入做加权求和再过激活函数 | `z = Σ(wᵢxᵢ) + b`，`a = f(z)` |
| **激活函数** | 引入非线性，让网络能学习复杂模式 | 隐藏层首选 ReLU/GELU，输出层按任务选 |
| **ReLU** | `max(0, x)`，正区间梯度恒为 1 | 解决梯度消失，计算快，是默认选择 |
| **Sigmoid** | 压缩到 (0, 1)，可解释为概率 | 二分类输出层用，隐藏层不推荐（梯度消失） |
| **GELU** | `x·Φ(x)`，平滑版 ReLU | Transformer/LLaMA 隐藏层标配 |
| **全连接层** | `nn.Linear(in, out)`，每个输入连接每个输出 | 基本构建块，参数量 = in×out + out |
| **前向传播** | 数据从输入层到输出层的计算过程 | `output = model(input)` |
| **反向传播** | 用链式法则计算每个参数的梯度 | `loss.backward()` 自动完成 |
| **损失函数** | 衡量预测与真实值的差距 | 分类用 CrossEntropy，回归用 MSE |
| **梯度下降** | `w = w - lr × ∂Loss/∂w` | 学习率太大震荡，太小收敛慢 |
| **Adam** | 自适应学习率的优化器 | 通用默认选择，大模型用 AdamW |
| **过拟合** | 训练集好但测试集差 | 用 Dropout/正则化/早停/数据增强解决 |
| **Dropout** | 训练时随机关闭神经元 | 防止过拟合，推理时自动关闭 |
| **BatchNorm** | 归一化每层输入 | 稳定训练、加速收敛 |
| **早停法** | 验证集不提升时停止训练 | 最简单有效的防过拟合手段 |
| **评估指标** | accuracy/Precision/Recall/F1 | 漏报代价高用 Recall，兼顾用 F1 |
| **PyTorch** | 深度学习框架（当前 2.13.0） | 动态图、Pythonic、研究+生产首选 |
| **Tensor** | 多维数组，GPU 可加速 | `torch.FloatTensor`、`.to(device)` |
| **DataLoader** | 批量加载数据 | `batch_size=32`、`shuffle=True` |
| **标准化** | z-score：`(x - mean) / std` | 训练前必须做，让各特征在同一量级 |

## 五、本章面试题

### 题目 1：什么是梯度消失问题？ReLU 是如何缓解的？

**难度**：⭐⭐  
**类型**：核心概念

**参考答案**：

梯度消失是指在深层网络的反向传播中，梯度经过多层连乘后变得极小（接近 0），导致浅层参数几乎无法更新。Sigmoid 函数的导数最大值仅为 0.25（在 x=0 处），多层连乘后梯度指数级衰减——例如 10 层网络的梯度最多为 0.25^10 ≈ 0.0000009。ReLU 在正区间导数恒为 1，多层连乘不会衰减，完美解决了这个问题。但 ReLU 有"死亡神经元"问题——当输入为负时导数为 0，该神经元永远不再更新。解决方案包括 Leaky ReLU（负区间给小斜率 0.01）和 GELU（平滑过渡）。在大模型（Transformer）中，GELU 和 SiLU/Swish 逐渐替代 ReLU 成为主流选择。

---

### 题目 2：为什么深度学习需要激活函数？如果全部用线性激活会怎样？

**难度**：⭐  
**类型**：基础原理

**参考答案**：

如果没有非线性激活函数，无论网络有多少层，整个网络都等价于一个线性变换 `y = Wx + b`（矩阵乘法的嵌套还是矩阵乘法）。这意味着深层网络退化为单层线性模型，完全丧失了学习复杂模式的能力。在养老院场景中，如果用线性模型评估健康风险，只能学到"指标越高风险越大"这种线性关系，无法学到"心率太高或太低都危险"这种 U 形关系，也无法学到"血糖略高但血氧正常时风险不大，但两者同时异常时风险骤增"这种交互效应。非线性激活函数让神经网络成为"万能函数逼近器"——理论上足够深的网络可以拟合任意连续函数。

### 题目 3：比较 Batch Normalization 和 Layer Normalization，它们分别用在什么场景？

**难度**：⭐⭐  
**类型**：架构设计

**参考答案**：

Batch Norm（BN）和 Layer Norm（LN）的核心区别在于归一化的维度不同。BN 沿 batch 维度归一化——对同一个特征，在一个 batch 内的所有样本上计算均值和方差。LN 沿特征维度归一化——对同一个样本，在所有特征上计算均值和方差。使用场景：① BN 依赖 batch 统计量，batch_size 太小时统计不稳定，因此 BN 适合 CV 任务（batch_size 通常 32-256）；② LN 不依赖 batch，适合 NLP/Transformer 任务（序列长度可变，batch_size 通常较小）；③ 所有 Transformer 架构（GPT、LLaMA、BERT）都用 Layer Norm。在养老院场景中，如果每次只评估 1 位长者（batch_size=1），BN 无法计算统计量，LN 仍然正常工作。

### 题目 4：Adam 优化器的原理是什么？它比 SGD 好在哪里？

**难度**：⭐⭐  
**类型**：优化算法

**参考答案**：

Adam（Adaptive Moment Estimation）结合了两个思想：① 动量（Momentum）——用梯度的指数移动平均来平滑更新方向，类似小球滚下山坡时的惯性；② 自适应学习率——为每个参数维护独立的学习率，梯度大的参数学习率自动减小，梯度小的参数学习率自动增大。具体地，Adam 维护两个移动平均：一阶矩 m（梯度均值）和二阶矩 v（梯度平方均值），更新时用 `m/√v` 做自适应调整。相比 SGD：Adam 收敛更快（自适应学习率省去了手动调参的麻烦）、对超参数更鲁棒（默认 lr=0.001 通常就能用）、能处理稀疏梯度。但 SGD + Momentum 在某些任务上泛化更好（图像分类），这可能是由于 Adam 的自适应学习率导致"尖锐最小值"。实践中，CV 任务可以先试 SGD+Momentum，NLP/Transformer 任务直接用 AdamW。

### 题目 5：什么是 Dropout？为什么它能防止过拟合？推理时怎么处理？

**难度**：⭐  
**类型**：正则化

**参考答案**：

Dropout 在训练时以概率 p 随机将神经元的输出置为 0。它防止过拟合的机制有几种解释：① 集成效应——每次 Dropout 产生不同的子网络，相当于训练了指数级数量的模型并做平均；② 减少共适应——迫使每个神经元独立工作，不能依赖某个特定的"搭档"神经元；③ 噪声注入——相当于给训练过程添加噪声，增强了模型的鲁棒性。推理时，Dropout 层不做任何丢弃，但将所有输出乘以 `(1-p)` 做缩放，以保持输出的期望值与训练时一致。PyTorch 的 `nn.Dropout` 在 `model.eval()` 时自动禁用，在 `model.train()` 时自动启用，开发者只需记得在训练和评估时切换模式。养老院类比：每次考核随机抽调不同的护理团队，避免整个团队只靠一个"王牌护理员"。

### 题目 6：解释 Precision、Recall、F1 Score 的含义和区别，以及在医疗场景中如何选择？

**难度**：⭐⭐  
**类型**：评估指标

**参考答案**：

- **Precision（精确率）**：预测为阳性的样本中，真正阳性的比例。`Precision = TP / (TP + FP)`。含义："模型说需要就医的人中，确实需要就医的有多少。"
- **Recall（召回率）**：真正阳性的样本中，被模型找到的比例。`Recall = TP / (TP + FN)`。含义："所有真正需要就医的人中，模型找到了多少。"
- **F1 Score**：Precision 和 Recall 的调和平均数。`F1 = 2 × P × R / (P + R)`。

在养老院/医疗场景中的选择：① 跌倒检测、心率异常告警——**优先 Recall**，因为漏报（没检测到跌倒）的代价远高于误报（误报一次最多虚惊一场）；② 自动开具医嘱——**优先 Precision**，因为误诊的代价极高；③ 健康风险分级——**用 F1 Score 或加权 F1**，需要兼顾。当类别不平衡时（如"极高风险"只有 5% 的样本），macro-average F1 比 accuracy 更有意义。

### 题目 7：PyTorch 中 `model.train()` 和 `model.eval()` 的区别是什么？不切换会有什么后果？

**难度**：⭐  
**类型**：工程实践

**参考答案**：

`model.train()` 和 `model.eval()` 切换模型的运行模式，影响 Dropout 和 BatchNorm 的行为。`model.train()` 模式：Dropout 会随机丢弃神经元，BatchNorm 使用当前 batch 的统计量。`model.eval()` 模式：Dropout 不做任何操作（所有神经元都参与），BatchNorm 使用训练阶段累计的全局统计量（running mean/var）。不切换的后果：① 如果评估时忘了 `model.eval()`，Dropout 仍在随机丢弃，导致每次预测结果不同（不确定性），且输出偏小（因为部分神经元被关闭）；② 如果训练时用了 `model.eval()`，Dropout 不工作，模型容易过拟合，BatchNorm 行为异常。最佳实践：在训练循环中严格配对使用——训练阶段用 `model.train()`，验证/测试/推理阶段用 `model.eval()`。

### 题目 8：什么是学习率预热（Warmup）？为什么大模型训练需要它？

**难度**：⭐⭐⭐  
**类型**：训练技巧

**参考答案**：

学习率预热是指在训练初期，让学习率从一个很小的值逐步增大到目标值，而不是一开始就用大学习率。原理：训练初期，模型参数是随机初始化的，梯度方向不可靠。如果一开始就用大学习率，可能导致参数大幅震荡，甚至发散。预热阶段让模型先用小步长"试探"，等梯度方向趋于稳定后再加大步长。大模型（如 LLaMA、GPT）训练特别需要 warmup 的原因：① 参数量巨大（数十亿），一个大梯度更新可能导致参数"跑偏"且难以恢复；② 使用 AdamW 优化器时，其自适应学习率在初期的估计不准确（二阶矩 v 的移动平均需要几百步才能稳定）；③ 分布式训练中，梯度同步的噪声在初期更大。典型设置：前 2000 步线性预热，然后余弦衰减。在养老院场景中，这就像新护理员入职时先从简单任务做起（warmup），逐步适应后再接手复杂工作。

### 题目 9：解释前向传播和反向传播的计算过程，以及 PyTorch 如何自动求导。

**难度**：⭐⭐  
**类型**：核心原理

**参考答案**：

**前向传播**：输入数据从第一层逐层传递到最后一层，每层做线性变换（`y = Wx + b`）加激活函数。过程中 PyTorch 会构建一个**计算图**（Computational Graph），记录每一步操作及其输入，以便后续求梯度。**反向传播**：从损失函数开始，利用链式法则（Chain Rule）沿着计算图反向逐层计算梯度。例如 `Loss = f(g(h(x)))`，则 `∂Loss/∂x = f'(g(h))) × g'(h(x)) × h'(x)`。PyTorch 使用**自动微分**（Autograd）机制实现自动求导：① 前向传播时，每个 Tensor 操作都会在计算图中记录节点；② 调用 `loss.backward()` 时，从 loss 节点开始反向遍历计算图，自动计算每个参数的梯度；③ 梯度存储在 `param.grad` 中，供优化器使用。`torch.no_grad()` 上下文管理器会禁用计算图构建，用于推理阶段节省内存。

### 题目 10：数据标准化（StandardScaler）为什么重要？如果不做会怎样？

**难度**：⭐  
**类型**：数据预处理

**参考答案**：

数据标准化（z-score：`(x - μ) / σ`）将各特征转换为均值 0、标准差 1 的分布。不做标准化的影响：① **梯度下降效率低**——不同特征的量级差异大（心率 60-120 vs 血氧 85-100），损失函数的等高线呈椭圆形，梯度方向偏离最优方向，导致收敛缓慢或震荡；② **权重不均衡**——大量级特征（如血压 120-180）的权重会被迫调得很小，小量级特征（如体温 36-39）的权重被迫调得很大，模型对数值敏感而非对特征含义敏感；③ **正则化失效**——L2 正则化对所有权重施加相同惩罚，但如果特征量级不同，大权重可能是合理的（对应小量级特征），被错误惩罚。在养老院场景中，心率的范围是 60-180，血氧的范围是 80-100——如果不标准化，模型会认为心率"更重要"（数值大），而实际上血氧低于 90 才是更危险的信号。注意：`scaler.fit_transform()` 只在训练集上 fit，验证/测试集用 `scaler.transform()`（使用训练集的均值和标准差），否则会造成数据泄露。

### 题目 11：什么是类别不平衡？如何处理？请举例说明。

**难度**：⭐⭐  
**类型**：实际问题

**参考答案**：

类别不平衡是指训练数据中各类别的样本数量差异悬殊。在养老院场景中极为常见——"低风险"长者占大多数（60-70%），"极高风险"可能只有 5-10%。如果不处理，模型会倾向于把所有样本预测为多数类（"全部预测为低风险"也能达到 70% 准确率）。处理方法：① **加权损失函数**——给少数类更高的权重。`nn.CrossEntropyLoss(weight=torch.tensor([1.0, 1.5, 2.0, 4.0]))`，"极高风险"的权重最大；② **过采样**——复制少数类样本或用 SMOTE 生成合成样本；③ **欠采样**——减少多数类样本（数据量大时可用）；④ **Focal Loss**——自动降低易分类样本的权重，聚焦于难分类的少数类样本；⑤ **分层采样**——`train_test_split(..., stratify=y)` 确保各集合类别比例一致。实际项目中推荐①+⑤组合使用。

### 题目 12：解释 PyTorch 的动态计算图与 TensorFlow 的静态计算图的区别，以及 PyTorch 为什么更受欢迎。

**难度**：⭐⭐⭐  
**类型**：框架对比

**参考答案**：

计算图是深度学习框架记录运算过程的数据结构，用于自动求导。**静态图**（TensorFlow 1.x 风格）：先定义完整的计算图，再送入数据执行。优点是可优化（编译期优化、图剪枝），缺点是调试困难（不能用 Python 原生调试器打断点），代码不直观。**动态图**（PyTorch 风格）：每次前向传播时实时构建计算图，用完即弃。优点是符合 Python 的直觉编程方式（像写普通 Python 一样写神经网络），可以用 `print()`、`pdb`、断点调试，支持动态控制流（if/else、循环）。PyTorch 更受欢迎的原因：① 开发体验好——Python 原生风格，学习曲线低；② 调试方便——可以直接看中间变量；③ 研究社区首选——几乎所有最新论文都用 PyTorch 发布代码；④ 生产工具完善——`torch.compile`（2.0+）补齐了静态图的性能优势，`torch.export` 支持导出静态图用于部署。养老院类比：动态图就像"实时手写病历"（边看边写），静态图像"先填模板再录入"（先定义再执行），前者更灵活。

   李沐老师的开源教材，PyTorch 版本，每章都有可运行的 Jupyter Notebook。强烈推荐从第 3 章（线性回归）开始跟做。

2. **3Blue1Brown《Neural Networks》系列视频**  
   YouTube 上最好的神经网络可视化讲解，用动画演示梯度下降和反向传播的直觉。

3. **PyTorch 官方教程：pytorch.org/tutorials/**  
   从 "60 Minute Blitz" 开始，快速上手 PyTorch 的 Tensor、Autograd、nn.Module。

4. **《Deep Learning》（Goodfellow 等著，花书）**  
   深度学习领域的"圣经"，适合在理解代码后再回头补理论。重点看第 6-8 章（深度前馈网络、正则化、优化）。

5. **d2l.ai/chapter_optimization/index.html**  
   优化算法专题，可视化各种优化器的收敛轨迹（SGD、Momentum、Adam 等）。
