## 二、核心知识点

### 2.1 什么时候需要微调？

#### 类比

养老院培训护理员有两种方式：

```
方式 1：给手册让她自己查（RAG）
  - 优点：随时更新手册内容，不需要重新培训
  - 缺点：每次都要查手册，速度慢；手册里没有的就不会

方式 2：系统培训，把知识"刻在脑子里"（微调）
  - 优点：不用查手册就能回答，速度快；能处理手册里没有的情况
  - 缺点：培训成本高；知识更新需要重新培训
```

#### RAG vs 微调选择指南

| 维度 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 更新文档即可（分钟级） | 需要重新训练（小时级） |
| 响应速度 | 需要检索步骤（较慢） | 直接生成（较快） |
| 幻觉控制 | 强（有文档约束） | 弱（仍可能幻觉） |
| 成本 | 低（不需要 GPU 训练） | 高（需要 GPU + 数据标注） |
| 风格控制 | 弱（模型用通用风格） | **强（可以学习特定风格）** |
| 领域能力 | 依赖检索质量 | **模型内化领域知识** |

**养老院推荐策略**：
1. **先 RAG**——成本低、见效快，覆盖 80% 场景
2. **再微调**——针对特定任务（如护理记录生成）微调小模型
3. **RAG + 微调**——微调后的模型 + RAG 检索 = 最佳效果

### 2.2 微调的核心概念

#### 全参数微调 vs 参数高效微调

```python
# 全参数微调（Full Fine-tuning）
# 更新模型的所有参数
# 7B 模型 → 7B 个参数需要更新 → 需要 4×A100（80GB）
# 成本：几百到几千美元
#
# 类比：让护理员重新上一遍护理学校（全部重学）

# 参数高效微调（PEFT, Parameter-Efficient Fine-Tuning）
# 只更新一小部分参数（0.1%-1%）
# 7B 模型 → 只更新 10M-100M 参数 → 单张 RTX 4090（24GB）即可
# 成本：几美元到几十美元
#
# 类比：给护理员做专项培训（只学养老院的特殊流程）
```

#### LoRA（Low-Rank Adaptation）

```python
import torch
import torch.nn as nn

# LoRA 的核心思想：
# 在原始权重矩阵 W 旁边加一个低秩分解的 ΔW
# W_new = W + ΔW = W + B × A
# 其中 B: (d, r), A: (r, d), r << d（r 是秩，通常 8-64）
#
# 类比：
# 原始模型的知识 = 一本 1000 页的护理教科书（W）
# LoRA 修改 = 一本 10 页的养老院补充手册（ΔW = B×A）
# 最终知识 = 教科书 + 补充手册（W + ΔW）
#
# 优势：
# 1. 参数量极少：10 页 vs 1000 页（1% 的参数）
# 2. 显存占用小：不需要存储完整的梯度
# 3. 可以叠加：多个 LoRA 适配器可以切换

class LoRALinear(nn.Module):
    """简化的 LoRA 线性层实现"""
    
    def __init__(self, original_layer: nn.Linear, rank: int = 8, alpha: float = 16):
        """
        Args:
            original_layer: 原始的线性层（冻结）
            rank: LoRA 的秩（越小参数越少，但表达能力越弱）
            alpha: 缩放因子（控制 LoRA 的影响强度）
        """
        super().__init__()
        self.original = original_layer
        
        # 冻结原始参数
        for param in self.original.parameters():
            param.requires_grad = False
        
        # LoRA 矩阵
        d_in = original_layer.in_features
        d_out = original_layer.out_features
        
        self.lora_A = nn.Parameter(torch.randn(d_in, rank) * 0.01)   # 降维
        self.lora_B = nn.Parameter(torch.zeros(rank, d_out))          # 升维（初始化为 0）
        self.scaling = alpha / rank
        
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 原始输出 + LoRA 输出
        original_output = self.original(x)
        lora_output = (x @ self.lora_A @ self.lora_B) * self.scaling
        return original_output + lora_output


# 示例
original = nn.Linear(4096, 4096)
lora_layer = LoRALinear(original, rank=8)

# 统计参数
original_params = sum(p.numel() for p in original.parameters())
lora_params = sum(p.numel() for p in lora_layer.lora_A) + sum(p.numel() for p in lora_layer.lora_B)

print(f"原始层参数量: {original_params:,}")   # 16,777,216
print(f"LoRA 参数量: {lora_params:,}")         # 65,536（仅 0.39%）
print(f"参数减少: {(1 - lora_params/original_params)*100:.1f}%")
```

#### QLoRA — 量化 + LoRA

```python
# QLoRA = 4-bit 量化 + LoRA
# 
# 原理：把原始模型量化为 4-bit（显存减少 75%），然后在量化模型上加 LoRA
#
# 显存对比（7B 模型）：
# - 全参数微调（FP16）：~56 GB → 需要 2×A100
# - LoRA（FP16）：~16 GB → 需要 1×A100 或 1×RTX 4090
# - QLoRA（4-bit）：~6 GB → 需要 1×RTX 4060 Ti！
#
# 类比：
# 全参数微调 = 重新印一本教科书
# LoRA = 加一本补充手册
# QLoRA = 把教科书缩印后 + 补充手册（省纸省空间）
```

### 2.4 使用 TRL + PEFT 微调

```python
# 安装：uv add trl peft datasets transformers bitsandbytes accelerate
# 版本：trl 1.7.1, peft 0.19.1, datasets 5.0.0, bitsandbytes 0.49.2

import torch
from datasets import Dataset
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType
from trl import SFTTrainer, SFTConfig

# ========== 第一步：加载基座模型 ==========

model_name = "Qwen/Qwen3-8B"   # 基座模型（8B 参数）

# 4-bit 量化配置（QLoRA）
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,                    # 4-bit 量化加载
    bnb_4bit_quant_type="nf4",            # NF4 量化类型（QLoRA 论文推荐）
    bnb_4bit_compute_dtype=torch.bfloat16, # 计算精度
    bnb_4bit_use_double_quant=True,        # 双重量化（进一步压缩）
)

# 加载模型和分词器
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    quantization_config=bnb_config,
    device_map="auto",
    trust_remote_code=True,
)
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)

print(f"模型加载完成: {model_name}")
print(f"模型参数量: {model.num_parameters():,}")

# ========== 第二步：配置 LoRA ==========

lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,      # 任务类型：因果语言模型
    r=16,                               # LoRA 秩（8-64，越大效果越好但参数越多）
    lora_alpha=32,                      # 缩放因子（通常 = 2×r）
    lora_dropout=0.05,                  # Dropout 防止过拟合
    target_modules=[                    # 应用 LoRA 的层
        "q_proj", "k_proj", "v_proj",   # 注意力层
        "o_proj", "gate_proj",           # 输出层和门控层
        "up_proj", "down_proj",          # FFN 层
    ],
    bias="none",
)

# 应用 LoRA
model = get_peft_model(model, lora_config)

# 打印可训练参数量
model.print_trainable_parameters()
# 输出：trainable params: 41,943,040 || all params: 8,072,204,288 || trainable%: 0.5196
# 只有 0.52% 的参数需要训练！

# ========== 第三步：准备数据集 ==========

def format_dataset(data: list[dict]) -> Dataset:
    """将数据转为训练格式"""
    formatted = []
    for item in data:
        text = f"""### 指令：
{item['instruction']}

### 输入：
{item['input']}

### 回答：
{item['output']}"""
        formatted.append({"text": text})
    return Dataset.from_list(formatted)

train_dataset = format_dataset(training_data)

# ========== 第四步：配置训练参数 ==========

training_config = SFTConfig(
    output_dir="./nursing_model_output",
    num_train_epochs=3,                    # 训练轮数（3-5 轮通常足够）
    per_device_train_batch_size=2,         # 每设备批次大小
    gradient_accumulation_steps=4,         # 梯度累积（等效 batch_size=8）
    learning_rate=2e-4,                    # 学习率（LoRA 通常用 1e-4 到 3e-4）
    warmup_ratio=0.03,                     # 预热比例
    lr_scheduler_type="cosine",            # 学习率调度器
    logging_steps=10,                      # 每 10 步打印日志
    save_strategy="epoch",                 # 每轮保存一次
    bf16=True,                             # 使用 bfloat16 精度
    max_seq_length=2048,                   # 最大序列长度
    gradient_checkpointing=True,           # 梯度检查点（节省显存）
)

# ========== 第五步：训练 ==========

trainer = SFTTrainer(
    model=model,
    args=training_config,
    train_dataset=train_dataset,
    processing_class=tokenizer,
)

# 开始训练
# trainer.train()

# 保存 LoRA 适配器
# trainer.save_model("./nursing_lora_adapter")
```

### 2.5 使用 Unsloth 加速微调

```python
# Unsloth 是 2026 年最流行的微调加速工具
# 核心优势：2-5 倍训练加速 + 60% 显存减少
# 版本：2026.7.2

# 安装：uv add unsloth

# 加载模型（Unsloth 自动优化）
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/Qwen3-8B",      # Unsloth 优化版模型
    max_seq_length=2048,
    dtype=None,                           # 自动选择精度
    load_in_4bit=True,                    # 4-bit 量化

# 添加 LoRA（Unsloth 简化版）

    model,
    r=16,

                     "gate_proj", "up_proj", "down_proj"],
    lora_alpha=32,
    lora_dropout=0,
    bias="none",
    use_gradient_checkpointing="unsloth",  # Unsloth 优化的梯度检查点
    random_state=42,

        output_dir="./output",

        gradient_accumulation_steps=4,
        num_train_epochs=3,
        learning_rate=2e-4,
        bf16=True,
        logging_steps=10,
    ),
```

## 四、知识点总结

| 概念 | 一句话解释 | 关键要点 |
|------|-----------|---------|
| **微调** | 用领域数据继续训练模型 | 让模型"内化"养老院的评估逻辑和格式 |
| **LoRA** | 低秩适配，只训练 0.5% 参数 | 冻结原始权重，加低秩矩阵 ΔW=B×A |
| **QLoRA** | 4-bit 量化 + LoRA | 7B 模型只需 6GB 显存，RTX 4060 可运行 |
| **PEFT** | 参数高效微调框架 | HuggingFace 官方，LoRA/Prefix/P-Tuning |
| **TRL** | Transformer 强化学习库 | SFTTrainer 简化监督微调流程 |
| **Unsloth** | 微调加速工具 | 2-5 倍加速，60% 显存减少 |
| **SFT** | 监督微调（Supervised Fine-Tuning） | 指令-回答对训练，最常用的微调方式 |
| **数据质量** | 质量 >> 数量 | 100 条高质量 > 10000 条低质量 |
| **NF4** | 4-bit NormalFloat 量化 | QLoRA 论文推荐的量化类型 |
| **梯度检查点** | 用计算换显存 | 训练时只保存部分激活值，需要时重新计算 |
| **模型合并** | 将 LoRA 权重写入基座 | merge_and_unload()，部署时只需要一个模型 |

## 五、本章面试题

### 题目 1：LoRA 的原理是什么？为什么它能用极少的参数达到接近全参数微调的效果？

**难度**：⭐⭐  
**类型**：核心原理

**参考答案**：

LoRA 的核心假设：模型微调时的权重变化矩阵 ΔW 是**低秩**的——虽然 ΔW 的维度与原始权重 W 相同（如 4096×4096），但它实际的有效维度远低于此。LoRA 将 ΔW 分解为两个低秩矩阵的乘积：ΔW = B × A，其中 B: (d, r), A: (r, d), r << d（r 通常为 8-64）。这样原本需要 d×d 个参数的 ΔW，只需要 d×r + r×d = 2dr 个参数。例如 d=4096, r=16 时，参数量从 16M 降到 131K（减少 99.2%）。效果接近全参数微调的原因：微调的任务通常是"领域适配"而非"从头学习"——模型的通用知识（语言能力、世界知识）已经足够好，只需要在特定领域做小幅调整，这些调整确实是低秩的。

---

### 题目 2：QLoRA 相比 LoRA 有什么改进？NF4 量化是什么？

**难度**：⭐⭐  
**类型**：技术细节

**参考答案**：

QLoRA 的改进：在 LoRA 的基础上，将基座模型从 FP16（16-bit）量化到 NF4（4-bit），显存减少 75%。具体技术：① **NF4（NormalFloat 4-bit）**——一种专为正态分布数据设计的 4-bit 量化格式，比传统的 INT4 量化精度更高，因为神经网络的权重分布近似正态，NF4 的量化点分布与之匹配；② **双重量化（Double Quantization）**——对量化常数本身再做一次量化，进一步节省显存；③ **分页优化器（Paged Optimizer）**——当显存不足时自动将优化器状态转移到 CPU 内存。效果：7B 模型全参数微调需要 ~56GB 显存，QLoRA 只需 ~6GB，单张 RTX 4060 Ti（8GB）即可运行。精度损失极小（<1%），在大多数任务上与 FP16 LoRA 效果相当。

---

### 题目 3：如何为养老院场景准备高质量的微调数据集？

**难度**：⭐⭐  
**类型**：数据工程

**参考答案**：

数据来源：① **历史护理记录**——从数据库导出经过审核的护理记录，确保质量；② **专家标注**——请护理主管、药剂师等专业人员标注评估结果；③ **模板生成**——基于标准模板 + 变量替换，批量生成训练数据。质量要求：① 准确性——医学信息必须正确，错误数据会导致模型学到错误知识；② 一致性——同类问题的输出格式和风格要统一；③ 多样性——覆盖不同疾病、不同风险等级、不同场景；④ 安全底线——涉及用药的回答必须有"请咨询医生"的免责。规模建议：200-500 条高质量数据即可显著提升效果。比数量更重要的是**数据的覆盖度**——宁可 200 条覆盖所有场景，也不要 2000 条都是相似场景。

### 题目 4：全参数微调、LoRA、Prompt Tuning 三种微调方式有什么区别？

**难度**：⭐⭐  
**类型**：技术对比

**参考答案**：

| 维度 | 全参数微调 | LoRA | Prompt Tuning |
|------|-----------|------|---------------|
| 更新参数 | 全部（100%） | 0.1-1% | <0.01% |
| 显存需求 | 极高（4×A100） | 中（1×A100） | 低（1×RTX 4090） |
| 训练成本 | 高 | 中 | 低 |
| 效果 | 最好 | 接近全参数 | 依赖任务复杂度 |
| 适用场景 | 大规模数据、新语言 | 领域适配 | 简单任务 |

养老院推荐：LoRA/QLoRA——在效果和成本之间取得最佳平衡。Prompt Tuning 对复杂任务（如护理评估报告生成）效果不够好，全参数微调成本太高。

### 题目 5：微调和 RAG 如何结合使用？

**难度**：⭐⭐  
**类型**：架构设计

**参考答案**：

微调 + RAG 是目前效果最好的组合方式：① **微调让模型"学会"输出格式和领域风格**——如养老院的护理记录格式、评估报告模板。这些不需要每次检索，微调后模型直接"记住"；② **RAG 提供最新的事实知识**——如长者的具体数据、最新的护理规范。这些需要实时检索，不能固化在模型中。结合方式：微调后的模型 + RAG 检索的知识 → 生成既符合格式又有最新信息的回答。养老院示例：模型微调后"知道"护理评估报告应该包含哪些部分、用什么格式，RAG 提供当前长者的具体数据和最新的护理指南。

### 题目 6：如何评估微调后的模型效果？

**难度**：⭐⭐  
**类型**：质量评估

**参考答案**：

评估维度：① **格式一致性**——微调后的模型是否按照预期格式输出（如护理记录的结构是否完整）；② **内容准确性**——评估结果是否医学正确（需要专家审核）；③ **安全合规**——是否包含"请咨询医生"等免责说明；④ **对比测试**——用相同的测试集比较微调前后的效果（BLEU/ROUGE 分数或人工评分）；⑤ **边界测试**——测试异常输入（空输入、超长输入、恶意输入）时模型的行为；⑥ **A/B 测试**——在真实用户中随机分配微调前后模型，比较用户满意度。养老院建议：准备 50-100 个测试用例的评估集，每次微调后跑一遍回归测试，确保效果提升且无退化。

### 题目 7：什么是过拟合？微调时如何防止过拟合？

**难度**：⭐  
**类型**：训练技巧

**参考答案**：

过拟合是指模型在训练数据上表现很好，但在新数据上表现差——模型"记住"了训练数据而不是"学会"了规律。防止过拟合的方法：① **数据量充足**——至少 200 条高质量数据，数据量越大越不容易过拟合；② **LoRA Dropout**——在 LoRA 层添加 Dropout（`lora_dropout=0.05`）；③ **减少训练轮数**——通常 3-5 轮即可，太多轮容易过拟合；④ **早停（Early Stopping）**——监控验证集损失，不再下降时停止训练；⑤ **学习率衰减**——使用 cosine 学习率调度器，后期学习率自动减小；⑥ **数据增强**——对训练数据做同义替换、改写等增强。养老院场景中，如果只有 200 条数据，建议训练 2-3 轮，LoRA rank 设为 8-16（不要太大）。

### 题目 8：Unsloth 的加速原理是什么？

**难度**：⭐⭐  
**类型**：优化技术

**参考答案**：

Unsloth 通过以下技术实现 2-5 倍训练加速：① **手动实现反向传播**——HuggingFace 的自动微分是通用的但不够高效，Unsloth 手动实现了 LoRA 层的反向传播，减少了不必要的计算；② **内核融合（Kernel Fusion）**——将多个小操作合并为一个 CUDA 内核，减少 GPU 调用开销；③ **优化的梯度检查点**——比 HuggingFace 默认的梯度检查点更高效，减少 60% 显存占用；④ **Flash Attention 集成**——自动使用 Flash Attention 2 加速注意力计算。效果：训练速度提升 2-5 倍，显存减少 60%。对养老院的实际意义：原本需要 A100 才能训练的 7B 模型，用 Unsloth 在 RTX 4060 Ti 上就能训练，硬件成本从数万元降到数千元。

### 题目 9：微调后的模型如何部署？

**难度**：⭐⭐  
**类型**：部署实践

**参考答案**：

部署流程：① **合并 LoRA**——用 `model.merge_and_unload()` 将 LoRA 权重合并回基座模型，生成一个完整的模型文件；② **格式转换**——转为 GGUF 格式（用于 Ollama）或保持 HF 格式（用于 vLLM）；③ **部署方式**——Ollama（本地，最简单）、vLLM（生产环境，高性能）、Docker 容器化部署；④ **API 服务**——用 FastAPI 或 vLLM 内置的 OpenAI 兼容 API 暴露服务。养老院推荐：合并后的模型用 Ollama 部署（本地运行，数据不出院），通过 OpenAI 兼容 API 接入 LangChain/LangGraph 系统。详见第 15 章。

### 题目 10：LoRA 的 rank 参数如何选择？

**难度**：⭐⭐  
**类型**：参数调优

**参考答案**：

rank（r）是 LoRA 的核心超参数，决定了适配器的"容量"。选择原则：① **r=8-16**——适合大多数微调任务，参数量少，训练快；② **r=32-64**——适合复杂任务或大领域差异（如从英文模型微调为中文护理模型）；③ **r=128+**——接近全参数微调的效果，但显存和训练时间增加。实验建议：先用 r=16 开始，如果效果不够再增大到 32 或 64。注意：r 增大时，lora_alpha 也应按比例增大（通常 alpha = 2×r）。养老院场景中，护理评估报告生成（需要学习特定格式）建议 r=16，通用对话风格适配建议 r=8。更大的 r 不一定更好——如果 r 太大，可能过拟合小数据集。
