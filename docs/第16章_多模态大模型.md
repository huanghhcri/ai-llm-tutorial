## 二、核心知识点

### 2.1 什么是多模态大模型？

#### 类比

养老院的医生看体检报告时：

```
纯文本模型 = 医生只看文字描述
  "血压 165/105mmHg，心率 95 次/分"
  → 能分析数据，但看不到图表趋势

多模态模型 = 医生直接看报告图片
  [体检报告图片]
  → 既能看到数字，也能看到心电图波形、趋势图表、医生手写备注
```

多模态大模型（VLM）能同时处理**文本 + 图片**（部分还支持音频/视频），将不同模态的信息融合理解。

### 2.3 本地部署多模态模型

```python
# 使用 transformers 加载 Qwen2.5-VL 模型
# 安装：uv add transformers torch pillow qwen-vl-utils

from transformers import Qwen2_5_VLForConditionalGeneration, AutoProcessor  # Qwen3-VL 使用相同的类
from qwen_vl_utils import process_vision_info
import torch

# ========== 加载模型 ==========
model_name = "Qwen/Qwen3-VL-8B-Instruct"  # 或 Qwen2.5-VL-7B-Instruct（更稳定）

# 加载模型（FP16，需要约 14GB 显存）
model = Qwen2_5_VLForConditionalGeneration.from_pretrained(
    model_name,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

# 加载处理器（包含 tokenizer 和 image processor）
processor = AutoProcessor.from_pretrained(model_name)

# ========== 推理 ==========
def analyze_image_local(image_path: str, question: str) -> str:
    """本地多模态推理"""
    
    # 构建消息
    messages = [
        {
            "role": "user",
            "content": [
                {"type": "image", "image": image_path},
                {"type": "text", "text": question},
            ],
        }
    ]
    
    # 处理输入
    text = processor.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
    image_inputs, video_inputs = process_vision_info(messages)
    inputs = processor(
        text=[text],
        images=image_inputs,
        videos=video_inputs,
        padding=True,
        return_tensors="pt",
    ).to(model.device)
    
    # 生成
    with torch.no_grad():
        generated_ids = model.generate(**inputs, max_new_tokens=1000)
    
    # 解码
    output = processor.batch_decode(generated_ids, skip_special_tokens=True)[0]
    return output
```

### 2.4 多模态 RAG

```python
# 多模态 RAG = 图片理解 + 文本检索 + 大模型生成
#
# 流程：
# 1. 文档中包含图片和文本
# 2. 图片用 VLM 转为文本描述
# 3. 文本和描述一起做 Embedding 存入向量数据库
# 4. 用户提问时，检索相关文本和图片
# 5. 将文本+图片一起发给多模态模型生成回答

from openai import OpenAI

client = OpenAI(api_key="sk-your-key", base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")


def extract_text_from_image(image_path: str) -> str:
    """用 VLM 从图片中提取文字描述"""
    import base64
    
    with open(image_path, "rb") as f:
        image_data = base64.b64encode(f.read()).decode("utf-8")
    
    response = client.chat.completions.create(
        model="qwen-vl-max",
        messages=[{
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{image_data}"}},
                {"type": "text", "text": "请详细描述这张图片的内容，提取所有可见的文字、数字和指标。"},
            ],
        }],
        temperature=0.1,
    )
    return response.choices[0].message.content


class MultimodalRAG:
    """
    多模态 RAG 系统。
    
    支持文本和图片的混合检索与生成。
    
    类比：护理员不仅能查文字档案，还能看体检报告图片、
    心电图波形、伤口照片等，综合所有信息给出建议。
    """
    
    def __init__(self, text_retriever, image_store: dict):
        self.text_retriever = text_retriever
        self.image_store = image_store    # {id: path} 图片存储
        self.image_descriptions = {}      # {id: description} 图片描述缓存
    
    def index_image(self, image_id: str, image_path: str):
        """索引图片：用 VLM 提取描述并存储"""
        if image_id not in self.image_descriptions:
            description = extract_text_from_image(image_path)
            self.image_descriptions[image_id] = description
        return self.image_descriptions[image_id]
    
    def query(self, question: str, include_images: bool = True) -> str:
        """
        多模态 RAG 查询。
        
        1. 检索相关文本
        2. 检索相关图片
        3. 将文本+图片发给多模态模型
        """
        # 检索相关文本
        text_docs = self.text_retriever(question)
        context = "\n\n".join([d["text"] for d in text_docs])
        
        if not include_images:
            # 纯文本模式
            return self._generate_text_only(question, context)
        
        # 检索相关图片（基于描述匹配）
        relevant_images = self._find_relevant_images(question)
        
        if not relevant_images:
            return self._generate_text_only(question, context)
        
        # 多模态生成
        return self._generate_multimodal(question, context, relevant_images)
    
    def _find_relevant_images(self, question: str, top_k: int = 2) -> list[str]:
        """根据问题找到相关图片"""
        # 简化实现：关键词匹配
        # 实际项目中用 Embedding 相似度检索
        relevant = []
        for img_id, desc in self.image_descriptions.items():
            # 检查描述中是否包含问题的关键词
            keywords = question.replace("？", "").replace("？", "").split()
            if any(kw in desc for kw in keywords if len(kw) > 1):
                relevant.append(img_id)
        return relevant[:top_k]
    
    def _generate_text_only(self, question: str, context: str) -> str:
        """纯文本生成"""
        response = client.chat.completions.create(
            model="qwen-plus",
            messages=[
                {"role": "system", "content": "你是养老院健康顾问。基于参考资料回答。"},
                {"role": "user", "content": f"参考资料：\n{context}\n\n问题：{question}"},
            ],
        )
        return response.choices[0].message.content
    
    def _generate_multimodal(self, question: str, context: str, image_ids: list[str]) -> str:
        """多模态生成"""
        import base64
        
        content = [{"type": "text", "text": f"参考资料：\n{context}\n\n问题：{question}"}]
        
        # 添加图片
        for img_id in image_ids:
            img_path = self.image_store.get(img_id)
            if img_path:
                with open(img_path, "rb") as f:
                    img_data = base64.b64encode(f.read()).decode("utf-8")
                content.append({
                    "type": "image_url",
                    "image_url": {"url": f"data:image/jpeg;base64,{img_data}"},
                })
        
        response = client.chat.completions.create(
            model="qwen-vl-max",
            messages=[
                {"role": "system", "content": "你是养老院健康顾问。综合文本资料和图片分析回答。"},
                {"role": "user", "content": content},
            ],
        )
        return response.choices[0].message.content
```

## 三、养老院业务实战案例

### 需求描述

构建一个**体检报告智能分析系统**：

1. 护理员上传体检报告图片
2. AI 提取报告中的所有指标
3. 对比历史数据，分析趋势
4. 生成健康评估报告和护理建议

### 完整代码

```python
"""
养老院体检报告智能分析 — 多模态实战
=====================================
第 16 章实战案例：用 VLM 分析体检报告图片

运行环境：Python 3.14
安装依赖：uv add openai pillow
"""

import json
import base64
from datetime import datetime
from typing import Optional
from openai import OpenAI


# ============================================================
# 配置
# ============================================================

# 使用 Qwen-VL（多模态模型）
vlm_client = OpenAI(
    api_key="sk-your-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

# 使用 Qwen-Plus（纯文本模型，用于分析和生成）
llm_client = OpenAI(
    api_key="sk-your-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)


# ============================================================
# 第一步：图片信息提取
# ============================================================

def extract_report_data(image_path: str) -> dict:
    """
    从体检报告图片中提取结构化数据。
    
    使用 VLM 识别图片内容，然后用 LLM 结构化。
    """
    # 读取图片
    with open(image_path, "rb") as f:
        image_data = base64.b64encode(f.read()).decode("utf-8")
    
    # 第一步：VLM 提取原始文本
    vlm_response = vlm_client.chat.completions.create(
        model="qwen-vl-max",
        messages=[{
            "role": "user",
            "content": [
                {
                    "type": "image_url",
                    "image_url": {"url": f"data:image/jpeg;base64,{image_data}"},
                },
                {
                    "type": "text",
                    "text": """请仔细阅读这份体检报告图片，提取以下信息：
1. 体检人姓名、性别、年龄
2. 所有检验指标的名称、数值、单位、参考范围
3. 异常指标（标注 ↑ 或 ↓）
4. 医生的诊断意见或建议

请逐项列出，不要遗漏任何指标。""",
                },
            ],
        }],
        temperature=0.1,
        max_tokens=2000,
    )
    
    raw_text = vlm_response.choices[0].message.content
    
    # 第二步：LLM 结构化
    llm_response = llm_client.chat.completions.create(
        model="qwen-plus",
        messages=[
            {
                "role": "system",
                "content": """将体检报告文本转为 JSON 格式。
输出格式：
{
    "patient": {"name": "姓名", "gender": "性别", "age": 年龄},
    "date": "体检日期",
    "indicators": [
        {"name": "指标名", "value": "数值", "unit": "单位", "reference": "参考范围", "status": "正常/偏高/偏低"}
    ],
    "abnormal_indicators": ["异常指标1", "异常指标2"],
    "doctor_advice": "医生建议"
}""",
            },
            {"role": "user", "content": raw_text},
        ],
        temperature=0.0,
        response_format={"type": "json_object"},
    )
    
    return json.loads(llm_response.choices[0].message.content)


# ============================================================
# 第二步：健康趋势分析
# ============================================================

def analyze_health_trend(
    current_data: dict,
    history: list[dict],
) -> str:
    """
    对比历史数据，分析健康趋势。
    
    Args:
        current_data: 当前体检数据
        history: 历史体检数据列表
    """
    prompt = f"""你是养老院的健康数据分析师。

当前体检数据（{current_data.get('date', '未知日期')}）：
{json.dumps(current_data.get('indicators', []), ensure_ascii=False, indent=2)}

历史体检数据：
"""
    for i, h in enumerate(history, 1):
        prompt += f"\n第{i}次（{h.get('date', '未知')}）：\n"
        prompt += json.dumps(h.get('indicators', []), ensure_ascii=False, indent=2)
    
    prompt += """

请分析：
1. 各项指标的变化趋势（上升/下降/稳定）
2. 需要关注的异常变化
3. 综合健康趋势评估（改善/稳定/恶化）
4. 下次体检建议关注的重点"""
    
    response = llm_client.chat.completions.create(
        model="qwen-plus",
        messages=[
            {"role": "system", "content": "你是健康数据趋势分析专家。用数据说话，给出具体的趋势判断。"},
            {"role": "user", "content": prompt},
        ],
        temperature=0.3,
    )
    return response.choices[0].message.content


# ============================================================
# 第三步：生成健康评估报告
# ============================================================

def generate_health_report(
    member_name: str,
    member_age: int,
    current_data: dict,
    trend_analysis: str,
) -> str:
    """生成完整的健康评估报告"""
    
    abnormal = current_data.get("abnormal_indicators", [])
    doctor_advice = current_data.get("doctor_advice", "")
    
    prompt = f"""你是养老院的健康评估专家。根据以下信息生成一份完整的健康评估报告。
```

## 五、本章面试题

### 题目 1：什么是 VLM？它和纯文本大模型有什么区别？

**难度**：⭐  
**类型**：基础概念

**参考答案**：

VLM（Vision-Language Model）是能同时理解图片和文本的多模态大模型。与纯文本模型的区别：① **输入模态**——纯文本模型只能处理文字，VLM 可以处理文字+图片（部分还支持音频/视频）；② **模型架构**——VLM 在 LLM 基础上增加了视觉编码器（如 ViT），将图片转为与文本对齐的向量表示；③ **能力差异**——VLM 能理解图片中的内容（文字、表格、图表、物体、场景），纯文本模型看不到图片。养老院场景中，VLM 能直接"看懂"体检报告图片、心电图波形、伤口照片，不需要先 OCR 转文字。

---

### 题目 2：如何通过 API 发送图片给多模态模型？

**难度**：⭐  
**类型**：API 使用

**参考答案**：

两种方式：① **图片 URL**——直接传图片的 HTTP 地址，最简单但需要图片可公网访问。格式：`{"type": "image_url", "image_url": {"url": "https://example.com/image.jpg"}}`；② **Base64 编码**——将图片文件读取后转为 Base64 字符串，适合本地图片。格式：`{"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,{base64_data}"}}`。注意事项：① 图片大小限制（通常 20MB 以内）；② Base64 编码后体积增大约 33%；③ 支持的格式：JPG、PNG、GIF、WebP；④ 多张图片可以在一条消息中发送。

---

### 题目 3：什么是多模态 RAG？它和纯文本 RAG 有什么区别？

**难度**：⭐⭐  
**类型**：架构设计

**参考答案**：

多模态 RAG 在纯文本 RAG 的基础上增加了图片处理能力。区别：① **索引阶段**——纯文本 RAG 只索引文字，多模态 RAG 还需要用 VLM 将图片转为文字描述后再索引；② **检索阶段**——除了文本向量检索，还需要图片相似度检索（基于图片描述或图片 Embedding）；③ **生成阶段**——纯文本 RAG 只发文字给 LLM，多模态 RAG 将文字+图片一起发给 VLM。养老院场景中，护理手册中包含伤口图片、心电图示例等——多模态 RAG 能检索到这些图片并一起发给模型分析。

---

### 题目 4：如何用 VLM 做 OCR（文字识别）？

**难度**：⭐  
**类型**：实践应用

**参考答案**：

VLM 的 OCR 能力比传统 OCR（如 Tesseract）更强——它不仅能识别文字，还能理解表格结构、图表含义。实现方式：将图片发给 VLM，Prompt 中要求"提取图片中的所有文字、数字和表格"。优势：① 能处理手写文字；② 能理解表格的行列结构；③ 能识别图表中的数据点；④ 支持多语言。养老院场景中，体检报告通常是 PDF 或拍照图片——用 VLM 提取指标名称、数值、参考范围、异常标记，比传统 OCR 准确率高得多。

---

### 题目 5：本地部署多模态模型需要什么硬件？

**难度**：⭐  
**类型**：硬件规划

**参考答案**：

取决于模型大小：① **Qwen2.5-VL-3B**——FP16 约 6GB 显存，INT4 约 3GB，RTX 4060（8GB）可运行；② **Qwen2.5-VL-7B**——FP16 约 14GB 显存，INT4 约 5GB，RTX 4060 Ti（8GB）INT4 可运行；③ **Llama-4-Maverick**——MoE 架构，激活参数 17B，需要更大显存。推荐：养老院场景用 Qwen2.5-VL-7B + INT4 量化，RTX 4060 Ti 即可运行，能处理体检报告、心电图等图片。

---

### 题目 6：多模态 Agent 的工作流程是什么？

**难度**：⭐⭐  
**类型**：Agent 设计

**参考答案**：

多模态 Agent 的工作流程：① **接收图片**——护理员拍照上传（如伤口照片、体检报告）；② **图片理解**——VLM 分析图片内容（伤口类型、报告指标）；③ **信息查询**——Agent 调用工具查询该长者的病历、用药、历史记录；④ **知识检索**——搜索护理知识库中的相关方案；⑤ **综合分析**——结合图片信息、长者数据、知识库，生成处理建议；⑥ **执行操作**——创建护理记录、发送通知、安排复查。养老院示例：护理员拍压疮照片 → Agent 识别为"2 期压疮" → 查询该长者有糖尿病（影响愈合） → 搜索压疮护理方案 → 生成个性化护理建议。

### 题目 7：Base64 编码图片有什么优缺点？

**难度**：⭐  
**类型**：技术细节

**参考答案**：

优点：① 自包含——图片数据嵌入在 JSON 中，不需要额外的文件传输；② 兼容性好——所有 API 和编程语言都支持；③ 安全——不需要暴露图片的 HTTP URL。缺点：① 体积增大——Base64 编码后体积增大约 33%；② 传输效率——大图片的 Base64 字符串很长，增加网络传输时间；③ 内存占用——编码和解码需要额外内存。优化建议：① 压缩图片（降低分辨率、质量）后再编码；② 小于 1MB 的图片用 Base64，大于 1MB 的用 URL；③ 批量处理时考虑用 URL 而非 Base64。

### 题目 8：如何评估多模态模型的图片理解能力？

**难度**：⭐⭐  
**类型**：质量评估

**参考答案**：

评估维度：① **文字识别准确率**——给模型一张体检报告图片，对比提取的文字与人工标注的准确率；② **表格结构理解**——检查模型是否正确理解了表格的行列关系；③ **指标提取完整性**——是否遗漏了报告中的任何指标；④ **异常检测**——是否正确识别了异常指标；⑤ **趋势分析**——对比多份报告时，是否正确识别了趋势。养老院建议：准备 50 张不同类型的体检报告图片，人工标注后作为测试集，评估模型的提取准确率。

### 题目 9：多模态模型和传统 OCR + LLM 的方案有什么区别？

**难度**：⭐⭐  
**类型**：方案对比

**参考答案**：

| 维度 | 传统 OCR + LLM | 多模态模型（VLM） |
|------|---------------|-------------------|
| 流程 | OCR 提取文字 → LLM 分析 | VLM 直接看图分析 |
| 准确率 | 取决于 OCR 质量 | 端到端，通常更高 |
| 表格理解 | 弱（OCR 难以理解结构） | 强（直接理解表格） |
| 图表理解 | 无法处理 | 能理解趋势图、心电图 |
| 手写识别 | 较差 | 较好 |
| 成本 | 低（OCR 免费） | 较高（VLM API 更贵） |
| 延迟 | 两步处理，较慢 | 一步处理，较快 |

推荐：简单的文字提取用 OCR + LLM（成本低），复杂的报告理解（含表格、图表）用 VLM（效果好）。养老院体检报告通常包含表格和图表，推荐 VLM 方案。

**难度**：⭐⭐⭐

集成方案：① **图片上传**——前端通过 ASP.NET Core Web API 上传体检报告图片，保存到文件服务器或 OSS；② **调用 VLM**——ASP.NET Core 后端将图片 URL 或 Base64 发送给 Python 多模态服务（FastAPI），或直接调用 Qwen-VL API；③ **结果处理**——解析 VLM 返回的 JSON 结构化数据，存入数据库；④ **报告展示**——前端展示 AI 生成的健康评估报告，包含指标表格、趋势图表、护理建议；⑤ **流程**：护理员拍照上传 → ASP.NET Core 调用 VLM API → 提取指标 → 存入数据库 → 生成报告 → 前端展示。关键：图片压缩后再传输（减少延迟），异步处理（长报告生成可能需要 10-30 秒）。
