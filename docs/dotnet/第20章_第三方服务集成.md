## 1. HTTP 客户端封装（面试常问）

### 1.1 为什么不能直接 new HttpClient？

> **生活类比**：养老院的「专用电话线」——如果每个科室都拉一条独立电话线，很快电话线路就不够用了。正确的做法是共用一个电话总机。

`new HttpClient()` 有两个致命问题：

| 问题 | 说明 | 后果 |
|------|------|------|
| **Socket 耗尽** | 每个实例有独立连接池，短生命周期实例导致 TCP 端口耗尽 | 高并发时 `SocketException: Address already in use` |
| **DNS 刷新** | HttpClient 长连接不会感知 DNS 变更 | 第三方服务 IP 变更后，HttpClient 还连旧 IP |

### 1.2 IHttpClientFactory 解决方案

```csharp
// ❌ 错误写法：每次 new HttpClient
public class SmsService
{
    public async Task SendAsync(string phone, string message)
    {
        var client = new HttpClient();  // 每次创建新实例，Socket 耗尽！
        await client.PostAsync("https://sms.aliyun.com/send", ...);
        client.Dispose();  // Dispose 后连接进入 TIME_WAIT
    }
}

// ✅ 正确写法：使用 IHttpClientFactory 类型化客户端
public class AliyunSmsClient
{
    private readonly HttpClient _httpClient;  // 由工厂管理生命周期

    public AliyunSmsClient(HttpClient httpClient)
    {
        _httpClient = httpClient;
        _httpClient.BaseAddress = new Uri("https://dysmsapi.aliyuncs.com");
    }

    public async Task SendSmsAsync(string phone, string templateCode, string param)
    {
        // 构建阿里云短信 API 请求...
        var response = await _httpClient.PostAsync("/sms/send", content);
        response.EnsureSuccessStatusCode();
    }
}

// Startup 注册
services.AddHttpClient<AliyunSmsClient>(client =>
{
    client.Timeout = TimeSpan.FromSeconds(10);
})
.AddTransientHttpErrorPolicy(p => p.WaitAndRetryAsync(3,
    retryAttempt => TimeSpan.FromSeconds(Math.Pow(2, retryAttempt))));
```

## 2. Polly 重试策略集成

### 2.1 为什么需要重试？

> **生活类比**：养老院前台打电话给家属——第一次占线，等 2 秒再打；第二次还占线，等 4 秒再打；第三次通了。

第三方服务可能因网络抖动临时失败，重试可以自动恢复。但不能无限重试，需要用 Polly 配置策略。

### 2.2 不同服务不同策略

```csharp
// 微信 API：指数退避重试（临时限流可能很快恢复）
services.AddHttpClient<WeChatClient>()
    .AddTransientHttpErrorPolicy(p => p
        .WaitAndRetryAsync(3, retryAttempt =>
            TimeSpan.FromSeconds(Math.Pow(2, retryAttempt))));

// 短信服务：固定间隔重试（防止短信重复发送）
services.AddHttpClient<AliyunSmsClient>()
    .AddTransientHttpErrorPolicy(p => p
        .WaitAndRetryAsync(2, _ => TimeSpan.FromSeconds(5)));

// OSS 上传：长超时 + 重试 1 次（大文件上传需要长超时）
services.AddHttpClient<AliyunOssClient>(client =>
{
    client.Timeout = TimeSpan.FromSeconds(30);
})
.AddTransientHttpErrorPolicy(p => p.RetryAsync(1));
```

## 3. 微信公众号/小程序 OAuth 登录流程

### 3.1 流程概览

```
家属打开小程序 → wx.login 获取 code
    → 前端将 code 发送到后端
    → 后端用 code + AppId + AppSecret 换取 openid
    → 查询 openid 是否已绑定长者账号
    → 已绑定：签发 JWT，返回登录成功
    → 未绑定：引导输入长者姓名+身份证后四位绑定
```

### 3.2 核心代码

```csharp
public class WeChatLoginAppService : ApplicationService
{
    private readonly IHttpClientFactory _httpClientFactory;
    private readonly WeChatOptions _options;
    private readonly IRepository<FamilyMember, Guid> _familyRepo;

    public async Task<WeChatLoginResultDto> LoginByCodeAsync(string code)
    {
        // 1. 用 code 换 openid
        var client = _httpClientFactory.CreateClient();
        var url = $"https://api.weixin.qq.com/sns/jscode2session" +
                  $"?appid={_options.AppId}&secret={_options.AppSecret}" +
                  $"&js_code={code}&grant_type=authorization_code";
        var resp = await client.GetStringAsync(url);
        var json = JsonDocument.Parse(resp);
        var openid = json.RootElement.GetProperty("openid").GetString();

        // 2. 查找是否已绑定
        var family = await _familyRepo.FirstOrDefaultAsync(f => f.OpenId == openid);
        if (family != null)
        {
            // 已绑定：签发 JWT
            return new WeChatLoginResultDto { Bound = true, Token = GenerateJwt(family) };
        }

        // 未绑定：返回 openid，引导绑定
        return new WeChatLoginResultDto { Bound = false, OpenId = openid };
    }
}
```

## 8. 自研微信小程序支付联动（财务场景）

### 8.1 与 §3 OAuth 登录的区别

§3 是家属绑定登录（`code` 换 `openid`，解决"你是谁"），§8 是账单缴费支付联动（订单生成、状态同步、回调，解决"你付了多少钱"）。

| 维度 | §3 微信 OAuth 登录 | §8 小程序支付联动 |
|------|-------------------|-------------------|
| 目的 | 身份认证，获取 openid | 账单缴费，资金流转 |
| 触发 | 家属打开小程序 | 家属点击"立即缴费" |
| 后端职责 | code → openid → 绑定账号 | 生成订单 → 推送银联 → 处理回调 |
| 密钥 | AppId + AppSecret | 银行/银联侧商户密钥（后端不持有） |

> **关键区别**：小程序端调起支付（`wx.requestPayment`）由银行/银联侧完成，后端不负责商户密钥管理。后端只做订单管理和回调处理。

### 8.2 接口清单（对照需求文档）

| 序号 | 接口 | 说明 | 触发时机 |
|------|------|------|---------|
| 1 | 账单查询推送 | 向小程序推送长者合同账单/长者聚合账单 | 财务审核后自动推送 |
| 2 | 预充值接口 | 展示余额、明细、触发充值 | 家属在小程序操作 |
| 3 | 支付联动 | **仅「真实付款发起后」才生成支付订单** | 浏览账单不生成订单 |
| 4 | 支付结果回调 | 成功/取消 → 更新订单状态、账单状态、预充值 | 银联异步通知 |
| 5 | 合并支付 | 多账单合并，后端须分摊到合同账单明细 | 家属选择多笔账单缴费 |

### 8.3 核心代码

```csharp
using System;
using System.Collections.Generic;
using System.ComponentModel.DataAnnotations;
using System.Linq;
using System.Threading.Tasks;
using Volo.Abp;
using Volo.Abp.Application.Services;

namespace Yls.Finance.Application.Contracts
{
    /// <summary>
    /// 小程序财务服务接口
    /// </summary>
    public interface IMiniProgramFinanceAppService : IApplicationService
    {
        /// <summary>向小程序推送长者账单（合同账单/聚合账单）</summary>
        Task PushBillAsync(PushBillInput input);
        /// <summary>创建支付订单（仅真实付款发起后才生成）</summary>
        Task<PaymentOrderDto> CreatePaymentOrderAsync(CreatePaymentOrderInput input);
        /// <summary>处理支付结果回调（幂等）</summary>
        Task HandlePaymentCallbackAsync(PaymentCallbackInput input);
    }

    /// 推送账单输入
    /// </summary>

    {
        [Required]
        public Guid ElderId { get; set; }
        /// <summary>账单类型：ContractBill / ElderBill</summary>

        /// <summary>账期（如 2026-07）</summary>
        public string? Period { get; set; }
    }

    /// <summary>
    /// 创建支付订单输入
    /// </summary>
    public class CreatePaymentOrderInput
    {
        [Required]
        public Guid ElderId { get; set; }
        /// <summary>要支付的合同账单明细 ID 列表（支持合并支付）</summary>
        [Required, MinLength(1)]
        public List<Guid> BillDetailIds { get; set; }
        /// <summary>支付渠道：BankUnionPay（银行银联）</summary>
        public string PayChannel { get; set; } = "BankUnionPay";
    }

    /// <summary>
    /// 支付结果回调输入（银行/银联异步通知）
    /// </summary>

        public string Status { get; set; }  // SUCCESS / CANCELLED

        public long AmountInCents { get; set; }
        public DateTime PayTime { get; set; }
        /// <summary>回调签名（复用第 17 章 timestamp+nonce+sign 方案）</summary>
        public string Timestamp { get; set; }
        public string Nonce { get; set; }
        public string Sign { get; set; }

    /// 支付订单 DTO

        /// <summary>支付渠道</summary>

        /// <summary>订单状态：Pending / Paid / Cancelled</summary>

// === 实现层 ===

namespace Yls.Finance.Application
{
    /// <summary>
    /// 小程序财务服务实现
    /// </summary>
    public class MiniProgramFinanceAppService
        : ApplicationService, IMiniProgramFinanceAppService
    {
        private readonly IRepository<ElderBill, Guid> _elderBillRepo;
        private readonly IRepository<ContractBillDetail, Guid> _billDetailRepo;
        private readonly IRepository<PaymentOrder, Guid> _orderRepo;
        private readonly IRepository<PrepaidAccount, Guid> _prepaidRepo;
        private readonly IBankPaymentGateway _bankGateway;  // 银行/银联网关

        public MiniProgramFinanceAppService(
            IRepository<ElderBill, Guid> elderBillRepo,
            IRepository<ContractBillDetail, Guid> billDetailRepo,
            IRepository<PaymentOrder, Guid> orderRepo,
            IRepository<PrepaidAccount, Guid> prepaidRepo,
            IBankPaymentGateway bankGateway)
        {
            _elderBillRepo = elderBillRepo;
            _billDetailRepo = billDetailRepo;
            _orderRepo = orderRepo;
            _prepaidRepo = prepaidRepo;
            _bankGateway = bankGateway;
        }

        /// <summary>
        /// 推送账单到小程序
        /// 财务审核后自动推送，或家属主动拉取
        /// </summary>
        public async Task PushBillAsync(PushBillInput input)
        {
            if (input.BillType == "ContractBill")
            {
                // 推送合同账单（真实财务依据）
                var bills = await AsyncExecuter.ToListAsync(
                    (await _billDetailRepo.GetQueryableAsync())
                        .Where(d => d.ElderId == input.ElderId)
                        .WhereIf(!string.IsNullOrEmpty(input.Period),
                            d => d.Period == input.Period));
                // 调用小程序服务号模板消息 / 订阅消息推送
            }
            else if (input.BillType == "ElderBill")
            {
                // 推送长者聚合账单（展示用，不作为财务依据）
                var elderBill = await _elderBillRepo.FirstOrDefaultAsync(
                    b => b.ElderId == input.ElderId);
                // 推送逻辑同上
            }
        }

        /// <summary>
        /// 创建支付订单
        /// 关键规则：仅「真实付款发起后」才生成订单，浏览账单不生成
        /// </summary>
        public async Task<PaymentOrderDto> CreatePaymentOrderAsync(CreatePaymentOrderInput input)
        {
            // 1. 校验账单明细
            var billDetails = new List<ContractBillDetail>();
            long totalAmount = 0;
            foreach (var detailId in input.BillDetailIds)
            {
                var detail = await _billDetailRepo.GetAsync(detailId);
                if (detail.IsPaid)
                {
                    throw new BusinessException("Finance.BillAlreadyPaid",
                        $"账单明细 {detailId} 已支付");
                }
                billDetails.Add(detail);
                totalAmount += (long)(detail.Amount * 100);  // 元转分
            }

            // 2. 生成订单编号
            var orderNo = $"PAY{DateTime.Now:yyyyMMddHHmmss}{Guid.NewGuid().ToString("N").Substring(0, 6).ToUpper()}";

            // 3. 创建支付订单
            var order = new PaymentOrder(
                GuidGenerator.Create(),
                orderNo,
                input.ElderId,
                totalAmount,
                input.PayChannel  // BankUnionPay
            );
            await _orderRepo.InsertAsync(order);

            // 4. 合并支付：分摊金额到合同账单明细
            // 需求：多账单合并支付，系统自动分摊每笔金额到具体账单明细
            foreach (var detail in billDetails)
            {
                detail.PaymentOrderId = order.Id;
                await _billDetailRepo.UpdateAsync(detail);
            }

            // 5. 推送订单到银行/银联，获取支付参数
            var bankResult = await _bankGateway.PushOrderAsync(new PaymentOrderDto
            {
                OrderNo = orderNo,
                TotalAmountInCents = totalAmount,
                PayChannel = input.PayChannel
            });

            // 6. 返回支付参数给小程序（小程序调起银联支付控件）
            return ObjectMapper.Map<PaymentOrder, PaymentOrderDto>(order);
        }

        /// <summary>
        /// 处理支付结果回调
        /// 关键规则：
        ///   - 幂等：同一 OrderNo 回调只处理一次
        ///   - 支付成功后状态为「待认领」，不直接改合同账单实收
        ///   - 取消预充值支付不生成账单
        /// </summary>
        public async Task HandlePaymentCallbackAsync(PaymentCallbackInput input)
        {
            // 1. 验签（复用第 17 章 timestamp+nonce+sign 方案）
            if (!VerifySignature(input))
            {
                Logger.LogWarning("支付回调验签失败: OrderNo={OrderNo}", input.OrderNo);
                throw new BusinessException("Finance.InvalidSignature", "回调签名验证失败");
            }

            // 2. 幂等：查询订单
            var order = await _orderRepo.FirstOrDefaultAsync(
                o => o.OrderNo == input.OrderNo);
            if (order == null)
            {
                Logger.LogWarning("收到未知订单回调: {OrderNo}", input.OrderNo);
                return;  // 幂等：未知订单忽略
            }

            // 3. 幂等：同一 OrderNo 回调只处理一次
            if (order.Status != "Pending")
            {
                Logger.LogInformation("订单 {OrderNo} 已处理（{Status}），跳过重复回调",
                    input.OrderNo, order.Status);
                return;
            }

            // 4. 更新订单状态
            if (input.Status == "SUCCESS")
            {
                order.Status = "PendingClaim";  // 待认领，不直接改合同账单实收
                order.BankTransactionNo = input.BankTransactionNo;
                order.PayTime = input.PayTime;

                // 注意：此处不更新合同账单明细的支付状态
                // 须等 T+1 银联划付流水 + 财务认领后才入账
            }
            else if (input.Status == "CANCELLED")
            {
                order.Status = "Cancelled";

                // 如果是预充值支付取消，不生成账单（需求原文）
                // 已关联的账单明细解除关联
                var details = await AsyncExecuter.ToListAsync(
                    (await _billDetailRepo.GetQueryableAsync())
                        .Where(d => d.PaymentOrderId == order.Id));
                foreach (var detail in details)
                {
                    detail.PaymentOrderId = null;
                    await _billDetailRepo.UpdateAsync(detail);
                }
            }

            await _orderRepo.UpdateAsync(order);
        }

        /// <summary>
        /// 回调验签（复用第 17 章接口签名验证方案）
        /// </summary>
        private bool VerifySignature(PaymentCallbackInput input)
        {
            // 按银行/银联指定方案验签
            // 通用方案：timestamp + nonce + sign = HMAC-SHA256(密钥, sorted params)
            var raw = $"{input.OrderNo}{input.Status}{input.Timestamp}{input.Nonce}";
            var expectedSign = ComputeHmacSha256(raw, _bankGateway.GetSignKey());
            return expectedSign == input.Sign;
        }

        private string ComputeHmacSha256(string data, string key)
        {
            using var hmac = new System.Security.Cryptography.HMACSHA256(
                System.Text.Encoding.UTF8.GetBytes(key));
            var hash = hmac.ComputeHash(System.Text.Encoding.UTF8.GetBytes(data));
            return BitConverter.ToString(hash).Replace("-", "").ToLower();
        }
    }
}
```

### 8.4 关键规则（需求原文）

| 规则 | 说明 |
|------|------|
| 线上必须全额缴费 | 不支持部分支付（线下缴费可选择部分明细） |
| 接口返回不可对同一笔支付重复入账 | 幂等校验：OrderNo + Status != Pending |
| 网络超时须提供手动补处理入口 | Dashboard 展示超时订单，支持财务手动确认 |

### 8.5 退款边界（硬性规则）

> ⚠️ **所有退款均不支持原路退回**，与《养老业财管理系统项目需求说明书》一致。

退款流程：退款申请 → 审批 → 生成退款单 → 同步金蝶 → 财务线下退款

**禁止在代码中出现**：
- `RefundToOriginalChannel`（原路退款方法）
- 微信退款 API 调用（`wx.refund`）
- 银行退款接口调用

覆盖场景：合同退款、预充值退款、无合同退款、押金退款。

## 9. 银行银联支付对接与 T+1 对账认领

### 9.1 架构定位

> **生活类比**：养老院的「对公账户收款核对」——家属转账到养老院对公账户，银行第二天把流水单给财务，财务逐笔核对"这笔钱是哪个长者的"。

合作银行提供银联收单能力（订单推送 + 划付流水 + 对账文件/接口）。养护平台推送订单至银行/银联 → 小程序完成支付 → T+1 获取划付流水 → 财务认领入账。

```
养护平台                    银行/银联                    家属小程序
    │                          │                          │
    │  1. 推送支付订单          │                          │
    │ ────────────────────────►│                          │
    │  ◄── 返回支付参数         │                          │
    │                          │  2. 调起支付控件           │
    │                          │ ◄────────────────────────│
    │                          │  3. 支付完成               │
    │                          │ ─────────────────────────►│
    │                          │                          │
    │  4. 异步回调（支付成功）    │                          │
    │ ◄────────────────────────│                          │
    │  5. 订单状态=PendingClaim  │                          │
    │                          │                          │
    │  ══════ T+1 日 ══════    │                          │
    │                          │                          │
    │  6. 拉取划付流水          │                          │

    │  ◄── 返回流水数据         │                          │
    │  7. 存入 BankSettlementFlow（状态=待认领）            │
    │                          │                          │
```

```csharp
/// <summary>
/// 银行支付网关实现（类型化 HttpClient + Polly 重试）
/// </summary>
public class BankPaymentGateway : IBankPaymentGateway, ITransientDependency
{
    private readonly HttpClient _httpClient;
    private readonly YlsFinanceOptions _options;

    public BankPaymentGateway(
        HttpClient httpClient,  // IHttpClientFactory 类型化客户端注入
        IOptions<YlsFinanceOptions> options)
    {
        _httpClient = httpClient;
        _options = options.Value;
    }

    /// <summary>
    /// 推送订单到银行/银联
    /// 配置：YlsFinanceOptions.BankPayment（ApiUrl、MerchantId、NotifyUrl）
    /// </summary>
    public async Task<BankPushResult> PushOrderAsync(PaymentOrderDto order)
    {
        var request = new
        {
            merchantId = _options.BankPayment.MerchantId,
            orderNo = order.OrderNo,
            amount = order.TotalAmountInCents,
            notifyUrl = _options.BankPayment.NotifyUrl,
            description = $"养老院账单-{order.OrderNo}"
        };

        var content = new StringContent(
            System.Text.Json.JsonSerializer.Serialize(request),
            System.Text.Encoding.UTF8,
            "application/json");

        // Polly 重试已在 IHttpClientFactory 注册时配置
        var response = await _httpClient.PostAsync(
            $"{_options.BankPayment.ApiUrl}/v1/orders", content);

        response.EnsureSuccessStatusCode();
        var result = System.Text.Json.JsonDocument.Parse(
            await response.Content.ReadAsStringAsync());

        return new BankPushResult
        {
            Success = result.RootElement.GetProperty("code").GetInt32() == 0,
            BankOrderNo = result.RootElement.GetProperty("bankOrderNo").GetString(),
            PayParams = System.Text.Json.JsonSerializer.Deserialize<Dictionary<string, string>>(
                result.RootElement.GetProperty("payParams"))
        };
    }

    public string GetSignKey() => _options.BankPayment.SignKey;

    /// <summary>
    /// 拉取 T+1 划付流水（调用银行对账 API）
    /// 由 Hangfire 每日凌晨 3 点调用
    /// </summary>
    public async Task<List<SettlementFlowDto>> GetSettlementFlowsAsync(DateTime date)
    {
        var url = $"{_options.BankPayment.ApiUrl}/v1/settlements?date={date:yyyy-MM-dd}";
        var response = await _httpClient.GetAsync(url);
        response.EnsureSuccessStatusCode();

        var json = System.Text.Json.JsonDocument.Parse(
            await response.Content.ReadAsStringAsync());

        var flows = new List<SettlementFlowDto>();
        foreach (var item in json.RootElement.GetProperty("data").EnumerateArray())
        {
            flows.Add(new SettlementFlowDto
            {
                FlowNo = item.GetProperty("flowNo").GetString(),
                Amount = item.GetProperty("amount").GetInt64(),
                Fee = item.GetProperty("fee").GetInt64(),
                SettlementDate = item.GetProperty("settlementDate").GetDateTime(),
                RelatedOrderNo = item.TryGetProperty("orderNo", out var orderNo)
                    ? orderNo.GetString() : null
            });
        }
        return flows;
    }
}
```

```csharp
/// <summary>
/// 接收银行/小程序支付结果异步通知
/// 关键：更新订单状态为「待认领」，不直接改合同账单实收
/// </summary>
public async Task HandleBankCallbackAsync(PaymentCallbackInput input)
{
    var order = await _orderRepo.FirstOrDefaultAsync(o => o.OrderNo == input.OrderNo);
    if (order == null || order.Status != "Pending") return;  // 幂等

    if (input.Status == "SUCCESS")
    {
        order.Status = "PendingClaim";  // 待认领，不直接改合同账单实收
        order.BankTransactionNo = input.BankTransactionNo;
        order.PayTime = input.PayTime;
    }
    else
    {
        order.Status = "Cancelled";
    }

    await _orderRepo.UpdateAsync(order);
}
```

> **为什么支付确认 ≠ 支付完成？** 银行 T+1 才划付资金到养老院对公账户。回调仅代表银行扣款成功，资金尚未到账。财务须等流水到账、认领确认后才算真正入账。

```csharp
public class BankSettlementReconciliationWorker

/// <summary>
/// 银行结算流水服务接口

public interface IBankSettlementService

/// <summary>
/// 银行结算流水服务实现
/// </summary>
public class BankSettlementService
    : ApplicationService, IBankSettlementService
{
    private readonly IRepository<BankSettlementFlow, Guid> _flowRepo;
    private readonly IRepository<ContractBillDetail, Guid> _billDetailRepo;
    private readonly IRepository<PrepaidAccount, Guid> _prepaidRepo;
    private readonly IBankPaymentGateway _bankGateway;
    private readonly IDistributedLock _distributedLock;  // Redis 分布式锁

    public BankSettlementService(
        IRepository<BankSettlementFlow, Guid> flowRepo,
        IRepository<ContractBillDetail, Guid> billDetailRepo,
        IRepository<PrepaidAccount, Guid> prepaidRepo,
        IBankPaymentGateway bankGateway,
        IDistributedLock distributedLock)
    {
        _flowRepo = flowRepo;
        _billDetailRepo = billDetailRepo;
        _prepaidRepo = prepaidRepo;
        _bankGateway = bankGateway;
        _distributedLock = distributedLock;
    }

    /// <summary>
    /// 拉取 T+1 划付流水
    /// 解析：流水号、划付金额、手续费、入账日期、关联订单号
    /// 存入 BankSettlementFlow 表，状态「待认领」
    /// </summary>
    public async Task PullDailyFlowsAsync(DateTime date)
    {
        var flows = await _bankGateway.GetSettlementFlowsAsync(date);

        foreach (var flow in flows)
        {
            // 幂等：检查流水号是否已导入
            var existing = await _flowRepo.FirstOrDefaultAsync(
                f => f.FlowNo == flow.FlowNo);
            if (existing != null) continue;

            var entity = new BankSettlementFlow(
                GuidGenerator.Create(),
                flow.FlowNo,
                flow.Amount,
                flow.Fee,
                flow.SettlementDate,
                flow.RelatedOrderNo
            )
            {
                Status = "待认领"
            };
            await _flowRepo.InsertAsync(entity);
        }

        Logger.LogInformation("银行划付流水同步完成：{Date}，共 {Count} 条",
            date.ToShortDateString(), flows.Count);
    }

    /// <summary>
    /// 认领流水
    /// 认领金额须与划付金额完全一致，不支持部分认领
    /// 认领后：更新合同账单明细支付状态 / 预充值余额
    /// </summary>
    public async Task ClaimFlowAsync(ClaimFlowInput input)
    {
        // Redis 分布式锁：防止同一流水重复认领
        await using var lockHandle = await _distributedLock.TryAcquireAsync(
            $"claim_flow:{input.FlowId}",
            TimeSpan.FromSeconds(30));

        if (lockHandle == null)
        {
            throw new BusinessException("Finance.FlowBeingClaimed",
                "该流水正在被其他用户认领，请稍后重试");
        }

        var flow = await _flowRepo.GetAsync(input.FlowId);

        if (flow.Status != "待认领")
        {
            throw new BusinessException("Finance.FlowAlreadyClaimed",
                "该流水已被认领或转入不明款");
        }

        // 认领金额须与划付金额完全一致
        if (input.ClaimAmount != flow.Amount)
        {
            throw new BusinessException("Finance.AmountMismatch",
                "认领金额必须与划付金额完全一致，不支持部分认领");
        }

        if (input.ClaimType == "线上支付认领")
        {
            var detail = await _billDetailRepo.GetAsync(input.BillDetailId!.Value);
            detail.IsPaid = true;
            detail.PaymentDate = flow.SettlementDate;
            detail.PaymentChannel = "银联";
            await _billDetailRepo.UpdateAsync(detail);

            // 生成财务收入明细（收款单）
            // ...
        }
        else if (input.ClaimType == "预充值认领")
        {
            var prepaid = new PrepaidAccount(
                GuidGenerator.Create(),
                input.ElderId!.Value,
                flow.Amount / 100m,
                "银行转账充值"
            );
            await _prepaidRepo.InsertAsync(prepaid);
        }

        flow.Status = "已认领";
        flow.ClaimedBy = CurrentUser.UserName;
        flow.ClaimedTime = Clock.Now;
        await _flowRepo.UpdateAsync(flow);
    }

    /// <summary>
    /// 转入不明款
    /// 认领错误 → 转不明款管理 → 再办理转款
    /// </summary>
    public async Task TransferToUnknownFundAsync(Guid flowId, string reason)
    {
        var flow = await _flowRepo.GetAsync(flowId);
        if (flow.Status != "待认领")
        {
            throw new BusinessException("Finance.InvalidOperation",
                "只有待认领流水可转入不明款");
        }

        flow.Status = "不明款";
        flow.Remark = reason;
        await _flowRepo.UpdateAsync(flow);
    }
}
```

## 10. 金蝶云·星空（K3 Cloud）业财同步

### 10.1 金蝶在养老院财务中的角色

> **生活类比**：养老院的「财务报表自动抄送」——业务系统记完账后，自动把数据同步到财务的总账系统，不用财务手动再录一遍。

业财一体：业务单据（账单/收款/退款）→ 金蝶财务凭证。不是简单 Excel 导出，而是 **API 推送 + 中间单拉取 + 双向校验**。

金蝶云·星空（K3 Cloud）是房地产/养老业财最常用的金蝶产品之一，支持多组织：`Location.JdCompany` 映射金蝶组织编码（如 `"08"`）。

### 10.2 五类单据触发时机（对照需求文档）

| 金蝶单据 | 业务触发 | 关键字段 | 财务含义 |
|---------|---------|---------|---------|
| **应收单** | 按应付日期聚合合同账单 | 含税金额、账期、`JdTypeCode` 科目 | 权责发生制下的应收确认 |
| **收款单** | 银行流水/银联流水认领完成后 | 流水号、手续费、实收金额 | 实际资金到账 |
| **摊销单** | 自然月日账累计（权责发生制） | 不含税摊销额、费项明细 | 按月确认服务收入，不论款项是否到账 |
| **转款单** | 预充值转合同账单/楼栋互转 | 转出/转入总额相等校验 | 资金性质变更 |
| **退款单** | 退款审批通过后（非原路退） | 线下退款执行后反写金蝶单据编号 | 不关联银行退款流水 |

> **摊销单 vs 收款单**（权责发生制 vs 收付实现制）：摊销单按服务时间确认收入（1 月住了就记 1 月收入），收款单按实际到账时间记账（1 月的费用 2 月才到账就记 2 月）。两者金额可能不一致。

```csharp
/// <summary>
/// 金蝶 K3 Cloud 同步服务实现
/// </summary>
public class KingdeeK3CloudSyncService
    : IKingdeeSyncService, ITransientDependency
{
    private readonly IKingdeeApiClient _apiClient;
    private readonly IRepository<SyncRecord, Guid> _syncRecordRepo;

    public KingdeeK3CloudSyncService(
        IKingdeeApiClient apiClient,
        IRepository<SyncRecord, Guid> syncRecordRepo)
    {
        _apiClient = apiClient;
        _syncRecordRepo = syncRecordRepo;
    }

    /// <summary>
    /// 同步应收单
    /// 按应付日期聚合合同账单，不是按服务时间
    /// </summary>
    public async Task SyncReceivableAsync(ContractBill bill)
    {
        // 1. 构建金蝶单据数据
        var data = KingdeeFieldMapper.MapToReceivableBill(bill);

        // 2. 调用金蝶 WebAPI（Save → Submit → Audit）
        var result = await _apiClient.SaveSubmitAuditAsync("AR_RECEIVEBILL", data);

        // 3. 本地记录同步状态（金蝶推送无本地 SyncStatus 记录是常见错误）
        var record = new SyncRecord(
            GuidGenerator.Create(),
            "应收单",
            bill.Id,
            result.Success ? "已同步" : "同步失败",
            result.KingdeeBillNo,
            result.ErrorMessage
        );
        await _syncRecordRepo.InsertAsync(record);

        if (!result.Success)
        {
            // 发布 CAP 事件，失败进 Hangfire 重推队列
            await EventBus.PublishAsync(new FinanceDocumentSyncEto
            {
                DocumentType = "应收单",
                DocumentId = bill.Id,
                ErrorMessage = result.ErrorMessage
            });
        }
    }

    public async Task SyncReceiptAsync(BankSettlementFlow flow)
    {
        var data = KingdeeFieldMapper.MapToReceiveBill(flow);
        var result = await _apiClient.SaveSubmitAuditAsync("AR_RECEIVEBILL", data);

        var record = new SyncRecord(
            GuidGenerator.Create(), "收款单", flow.Id,
            result.Success ? "已同步" : "同步失败",
            result.KingdeeBillNo, result.ErrorMessage);
        await _syncRecordRepo.InsertAsync(record);
    }

    public async Task SyncRefundAsync(RefundOrder refund)
    {
        var data = KingdeeFieldMapper.MapToRefundBill(refund);
        var result = await _apiClient.SaveSubmitAuditAsync("AR_REFUNDBILL", data);

        // 反写金蝶单据编号
        if (result.Success)
        {
            refund.KingdeeBillNo = result.KingdeeBillNo;
        }

        var record = new SyncRecord(
            GuidGenerator.Create(), "退款单", refund.Id,
```

```csharp
/// <summary>
/// 金蝶中间单拉取与双向校验
/// 定时从金蝶拉取中间单数据，与本地收款单/应收单比对
/// </summary>
public class KingdeeReconciliationWorker
    : AsyncPeriodicBackgroundWorkerBase
{
    private readonly IKingdeeApiClient _apiClient;
    private readonly IRepository<SyncRecord, Guid> _syncRecordRepo;

    protected override async Task DoWorkAsync(
        PeriodicBackgroundWorkerContext workerContext)
    {
        // 1. 从金蝶拉取中间单
        var kingdeeRecords = await _apiClient.GetIntermediateRecordsAsync();

        // 2. 与本地 SyncRecord 比对：金额、状态、单据编号
        var localRecords = await AsyncExecuter.ToListAsync(
            (await _syncRecordRepo.GetQueryableAsync())
                .Where(r => r.SyncStatus == "已同步"));

        foreach (var remote in kingdeeRecords)
        {
            var local = localRecords.FirstOrDefault(
                l => l.KingdeeBillNo == remote.BillNo);
            if (local == null)
            {
                // 金蝶有但本地无 → 告警
                Logger.LogWarning("金蝶单据 {BillNo} 在本地无对应记录", remote.BillNo);
                continue;
            }

            // 金额差异 → 告警
            if (Math.Abs(local.Amount - remote.Amount) > 0.01m)
            {
                Logger.LogWarning("金蝶单据 {BillNo} 金额不一致：本地={Local}，金蝶={Remote}",
                    remote.BillNo, local.Amount, remote.Amount);
            }
        }
    }
}
```

### 11.3 体征异常通知完整流程

```csharp
public class VitalSignAppService : ApplicationService
{
    private readonly IRepository<VitalSignRecord, long> _vitalRepo;
    private readonly IRepository<ElderProfile, long> _elderRepo;
    private readonly IRepository<FamilyMember, long> _familyRepo;
    private readonly IAliyunSmsService _smsService;
    private readonly IDingTalkService _dingTalkService;

    public async Task RecordVitalSignAsync(RecordVitalSignInput input)
    {
        var record = ObjectMapper.Map<RecordVitalSignInput, VitalSignRecord>(input);
        record.RecordTime = Clock.Now;
        record.RecordedBy = CurrentUser.Id.Value;
        await _vitalRepo.InsertAsync(record);

        var abnormalSigns = DetectAbnormalSigns(record);
        if (abnormalSigns.Any())
            await NotifyAbnormalAsync(record.ElderId, abnormalSigns);
    }

    private List<string> DetectAbnormalSigns(VitalSignRecord record)
    {
        var result = new List<string>();
        if (record.HeartRate > 120 || record.HeartRate < 50)
            result.Add($"心率异常：{record.HeartRate}次/分");
        if (record.SystolicPressure > 180 || record.DiastolicPressure > 110)
            result.Add($"血压过高：{record.SystolicPressure}/{record.DiastolicPressure}mmHg");
        if (record.Temperature > 38.5m) result.Add($"体温过高：{record.Temperature}℃");
        if (record.BloodOxygen < 90) result.Add($"血氧过低：{record.BloodOxygen}%");
        return result;
    }

    private async Task NotifyAbnormalAsync(long elderId, List<string> abnormalSigns)
    {
        var elder = await _elderRepo.GetAsync(elderId);
        var abnormalText = string.Join("；", abnormalSigns);

        // 短信通知家属
        var families = await _familyRepo.GetListAsync(
            f => f.BoundElderIds.Contains(elderId) && !string.IsNullOrEmpty(f.PhoneNumber));
        foreach (var family in families)
            await _smsService.SendSmsAsync(family.PhoneNumber, "SMS_20240001",
                new Dictionary<string, string>
                {
                    ["name"] = elder.Name,
                    ["sign"] = abnormalSigns.First().Split("：")[0],
                    ["value"] = abnormalSigns.First().Split("：").Last()
                });

        // 钉钉通知护理主管
        await _dingTalkService.SendWebhookMessageAsync("体征异常警报",
            $"## ⚠️ 体征异常警报\n- **长者**：{elder.Name}\n" +
            $"- **异常项**：{abnormalText}\n- **时间**：{Clock.Now:yyyy-MM-dd HH:mm}\n" +
            $"- **绑定家属已通知**：{families.Count} 人");
    }
}
```

## 常见错误与最佳实践

| # | 错误写法 | 正确写法 | 为什么错 |
|---|---------|---------|---------|
| 1 | 直接 `new HttpClient()` 使用 | 通过 `IHttpClientFactory` 创建 | Socket 耗尽 + DNS 不刷新，生产环境必崩 |
| 2 | HttpClient 超时不配置 | 每个客户端配置 `Timeout` 和 Polly 超时策略 | 第三方服务慢时拖垮整个应用 |
| 3 | 第三方 API 密钥硬编码在代码 | 存 appsettings.json 或环境变量，通过 Options 模式注入 | 密钥泄露到代码仓库，安全风险 |
| 4 | 短信/邮件发送失败不重试 | 配置 Polly 重试策略（指数退避，最多 3 次） | 临时网络抖动导致通知丢失 |
| 5 | 微信登录不校验 state 参数 | 生成随机 state 存 session，回调时校验 | CSRF 攻击，被伪造授权回调 |
| 6 | OSS 文件名用中文或特殊字符 | 用 GUID 重命名，原始文件名存数据库 | 编码不一致导致文件找不到 |
| 7 | 所有第三方服务用同一超时 | 不同服务配不同超时（微信 10s，短信 5s，OSS 30s） | 统一超时不合理，有的太短有的太长 |
| 8 | 第三方服务调用不做降级 | 配置 Polly 降级策略，失败时返回默认值或缓存 | 第三方宕机导致整个业务链路中断 |
| 9 | 支付回调直接改合同账单实收 | 回调只更新订单状态为「待认领」，等 T+1 认领后才入账 | 资金未到账就标记已收，账务与银行流水不一致 |
| 10 | 合并支付未分摊到账单明细 | 后端按比例分摊总金额到每笔合同账单明细 | 财务无法核对每笔明细的支付状态 |
| 11 | T+1 流水重复认领 | Redis 分布式锁 + 流水状态校验（待认领→已认领） | 同一笔资金重复入账，账务金额虚增 |
| 12 | 金蝶推送无本地 SyncStatus 记录 | 每次推送都写 SyncRecord（状态/单据编号/错误信息） | 同步失败无法追溯，无法重推 |
| 13 | 应收单按服务时间而非应付日期聚合 | 按应付日期聚合账单数据生成应收单 | 权责发生制下收入确认期间错误 |
| 14 | 线上支付退款调用银联/微信原路退款 API | 只走退款申请+审批+金蝶退款单+线下打款 | 需求明确不支持原路退回，原路退会导致账务与银行流水不一致 |

## 本章小结

| 知识点 | 核心要点 |
|--------|---------|
| HttpClient 问题 | Socket 耗尽 + DNS 刷新，必须使用 IHttpClientFactory |
| 类型化客户端 | 构造函数注入 HttpClient，框架管理生命周期和连接池 |
| Polly 重试 | WaitAndRetryAsync + 指数退避，不同服务配置不同策略 |
| 微信 OAuth | 授权码模式：wx.login → code → 后端换 openid → 生成 JWT |
| 阿里云短信 | SDK 封装 + 模板配置，体征异常自动通知家属 |
| 阿里云 OSS | 上传/下载/签名 URL，私有文件限时访问（呼应第15章） |
| MailKit | 异步发送 + 附件，月度健康报告邮件发送 |
| 钉钉对接 | 考勤 API + Webhook，护理员考勤同步排班系统 |
| 小程序支付 | 银联收单，T+1 划付流水认领后入账；支付确认 ≠ 支付入账 |
| 银行流水认领 | 导入银行流水 → 自动匹配订单 → 财务确认认领 → 更新账单状态 |
| 金蝶对接 | 五类单据同步（收款/应收/摊销/转款/退款），WebAPI Save/Submit/Audit |

## 面试题

### 面试题 1（中级 / 概念题）
**题目**：为什么不能直接 `new HttpClient()`？

**参考答案**：两个问题：①**Socket 耗尽**——每个 HttpClient 实例有独立连接池，大量短生命周期实例导致 TCP 端口耗尽（端口释放有 TIME_WAIT 期）；②**DNS 刷新**——HttpClient 长连接不会感知 DNS 变更，如果第三方服务 IP 变了，HttpClient 还连旧 IP。解决方案是 `IHttpClientFactory`，它管理 HttpClient 生命周期，复用 HttpMessageHandler，避免两个问题。养老院场景：调用微信 API 和阿里云 SMS 都必须用 IHttpClientFactory。

### 面试题 2（初级 / 概念题）
**题目**：IHttpClientFactory 有哪几种使用方式？

**参考答案**：三种：①**基本用法**——`_factory.CreateClient()` 创建默认客户端；②**命名客户端**——`_factory.CreateClient("WeChat")` 创建命名客户端，可为不同服务配置不同基地址和超时；③**类型化客户端**——定义 `WeChatClient` 类，构造函数注入 `HttpClient`，通过 DI 注册 `services.AddHttpClient<WeChatClient>()`。养老院场景：微信 API 用类型化客户端（`WeChatClient`），阿里云 SMS 用命名客户端（`AliyunSms`）。

### 面试题 3（高级 / 场景题）
**题目**：如何为不同的第三方服务配置不同的 Polly 策略？

**参考答案**：在 `AddHttpClient` 时链式配置 Polly。微信登录配置重试 3 次 + 指数退避（临时故障可恢复）；短信发送配置重试 2 次 + 熔断 5 次失败后断路 30 秒（防止短信服务商宕机拖垮系统）；OSS 上传配置超时 30 秒 + 重试 1 次（大文件上传需要长超时）。养老院场景：不同服务的可靠性不同，策略要差异化配置。

### 面试题 4（初级 / 概念题）
**题目**：微信扫码登录的完整流程是什么？

**参考答案**：OAuth2 授权码模式：①前端跳转微信授权页（带 AppId + RedirectUri + State）；②用户扫码确认；③微信回调带 code + state；④后端校验 state 防 CSRF；⑤用 code 换 access_token + openid；⑥用 openid 查是否已绑定系统账号；⑦已绑定签发 JWT，未绑定引导绑定。养老院场景：家属扫码绑定长者账号，查看亲人健康信息。

### 面试题 5（中级 / 概念题）
**题目**：阿里云短信发送需要注意什么？

**参考答案**：①模板必须提前审核通过，不能动态拼接内容；②签名必须是已审核的名称；③同一手机号每分钟最多 1 条、每天最多 10 条（防骚扰）；④发送失败要重试但不能无限重试；⑤敏感信息（验证码）不能写入日志。养老院场景：长者体征异常时短信通知家属，模板内容为"您的亲人XXX体征异常，请及时关注"。

### 面试题 6（中级 / 概念题）
**题目**：OSS 签名 URL 是什么？什么场景用？

**参考答案**：签名 URL（Presigned URL）是带临时授权的文件访问链接，有效期可配置（如 10 分钟）。客户端拿到签名 URL 后可直接访问 OSS 文件，无需经过应用服务器。场景：①前端直接上传大文件到 OSS（减轻应用服务器压力）；②生成临时下载链接给家属查看长者健康报告。养老院场景：月度健康报告生成后，生成 24 小时有效的签名 URL 发送给家属。

### 面试题 7（初级 / 概念题）
**题目**：MailKit 发送邮件的基本流程是什么？

**参考答案**：创建 `MimeMessage`（设置收件人/主题/正文）→ 创建 `Multipart`（添加正文和附件）→ 创建 `SmtpClient` → 连接 SMTP 服务器 → 认证 → 发送 → 断开。注意：①必须用 `using` 包裹 SmtpClient 确保连接释放；②异步发送用 `SendAsync`；③生产环境配置 SMTP 服务器地址和认证信息通过 Options 模式注入。养老院场景：每月自动发送费用明细邮件给家属。

### 面试题 8（高级 / 场景题）
**题目**：第三方服务宕机时如何做降级？

**参考答案**：Polly 降级策略：当重试和熔断都失败后，执行降级逻辑返回默认值。短信服务宕机时降级为"记录到数据库，服务恢复后补发"；微信登录宕机时降级为"提示用户稍后重试或使用手机号登录"。养老院场景：家属查看健康报告的 OSS 签名 URL 生成失败时，降级为"报告正在生成中，请稍后刷新"。

### 面试题 9（中级 / 概念题）
**题目**：钉钉考勤 API 对接的关键点是什么？

**参考答案**：①Access Token 有效期 2 小时，需要缓存并自动刷新；②考勤数据通过 Webhook 回调推送，需要配置回调地址并验证签名；③排班数据需要定时同步（每天凌晨拉取全量排班）。养老院场景：护理员排班在钉钉中管理，系统每天凌晨同步排班数据，考勤打卡数据实时回调同步到养老院系统。

### 面试题 10（高级 / 设计题）
**题目**：如何设计一个统一的第三方服务封装层？

**参考答案**：定义接口（如 `ISmsService`/`IEmailService`/`IFileStorageService`），每个接口有具体实现（`AliyunSmsService`/`MailKitEmailService`/`OssStorageService`）。通过 DI 注册，业务代码只依赖接口。配置通过 Options 模式注入。重试/超时/熔断策略在 `AddHttpClient` 时统一配置。养老院场景：如果从阿里云短信切换到腾讯云短信，只需新增 `TencentSmsService` 实现同一接口，业务代码零修改。

### 面试题 11（初级 / 概念题）
**题目**：Polly 的重试策略有哪些模式？

**参考答案**：三种：①**固定间隔**——每次重试间隔相同（如 2 秒）；②**指数退避**——每次重试间隔翻倍（2s→4s→8s），适合临时故障恢复；③**随机抖动**——在指数退避基础上加随机偏移，避免多客户端同时重试造成"惊群效应"。养老院场景：调用微信 API 用指数退避重试（临时限流可能很快恢复），调用短信服务用固定间隔重试。

### 面试题 12（高级 / 设计题）
**题目**：如何保证第三方服务调用的可观测性？

**参考答案**：三层保障：①**日志**——每次调用记录请求参数、响应状态、耗时（Serilog 结构化日志）；②**指标**——成功率、平均耗时、P99 耗时（Prometheus 指标）；③**告警**——成功率低于 95% 或 P99 超过 5 秒时触发告警。养老院场景：微信登录成功率下降到 90% 时自动告警，运维人员及时排查。

### 面试题 13（中级 / 场景题）
**题目**：养老院小程序支付成功后，为什么不能立即更新账单支付状态？

**参考答案**：因为小程序支付成功只代表银联扣款成功，资金尚未到达养老院对公账户。银联采用 T+1 划付机制，资金在次日才划付到养老院账户。财务需要先导入银行划付流水，再认领关联到对应账单明细，才算完成真正的入账。如果支付成功就更新账单状态，可能出现「账单显示已支付但资金未到账」的不一致情况。养老院场景：家属在小程序缴费后，账单状态显示「待确认」，等财务 T+1 认领后才变为「已支付」。

### 面试题 14（高级 / 设计题）
**题目**：如何设计银行流水认领的幂等性？

**参考答案**：银行流水认领的幂等性通过三层保障实现：①流水号唯一约束——数据库对 TransactionNo 建唯一索引，重复导入自动跳过；②认领状态校验——认领前检查 ClaimStatus 是否为「未认领」，已认领的流水拒绝重复操作；③金额校验——认领金额不得超过账单明细应付金额，防止超额认领。养老院场景：财务误操作重复导入同一批银行流水时，系统自动跳过已导入的流水，不会产生重复数据。

### 面试题 15（中级 / 概念题）
**题目**：金蝶云·星空 WebAPI 的 Save/Submit/Audit 三步分别做什么？

**参考答案**：**Save**（保存）将单据数据提交到金蝶数据库，状态为「暂存」，可以修改；**Submit**（提交）将暂存单据提交审核，状态变为「已提交」，不可直接修改；**Audit**（审核）确认单据生效，生成凭证，状态变为「已审核」。养老院场景：收款单先 Save 保存（可修改），再 Submit 提交（锁定），最后 Audit 审核（生成财务凭证）。如果审核失败，需要先反审核再修改。

### 面试题 16（高级 / 场景题）
**题目**：所有退款都不支持原路退回，这对系统设计有什么影响？

**参考答案**：不支持原路退回意味着系统不需要调用银行/银联的退款接口，退款流程简化为：生成退款单 → 审批 → 同步金蝶退款单 → 财务线下打款。设计影响：①退款单只需记录退款金额、收款人、退款原因，不需要记录银行退款流水号；②退款状态流转变为「待审批→已审批→已退款」，没有「退款中」状态；③退款确认由财务手动操作，系统只做记录和金蝶同步。养老院场景：家属申请退住退款，系统生成退款单，院长审批通过后，财务通过银行转账退款，再在系统中确认退款完成。

### 面试题 17（中级 / 概念题）
**题目**：合同账单和长者账单有什么区别？为什么要分开？

**参考答案**：合同账单是系统核心财务账单，基于入住合同自动生成，管理真实应收、分期明细和支付状态，是财务核算的依据。长者账单是面向家属展示的聚合账单，将合同账单和无合同账单按自然周期聚合，用于小程序展示和缴费。分开的原因：①合同账单有复杂的分期逻辑（首期/尾期/续签衔接），直接展示给家属太复杂；②长者账单需要聚合多个合同和无合同费用，简化为月度总额；③长者账单可以有多个版本（每次聚合生成新版本），合同账单只有一份。养老院场景：财务看合同账单做核算，家属看长者账单缴费。

### 面试题 18（高级 / 设计题）
**题目**：如何设计金蝶对接的容错机制？

**参考答案**：金蝶对接的容错设计包括：①重试机制——金蝶接口偶发超时，配置 Polly 指数退避重试 3 次；②幂等设计——每张单据携带唯一业务编号，金蝶侧用编号去重；③失败队列——同步失败的单据记录到失败队列，由定时任务定期重试；④手动补偿——Dashboard 展示同步失败的单据，支持财务手动重新推送；⑤session 管理——金蝶 session 有效期 30 分钟，客户端自动续期。养老院场景：凌晨同步收款单时金蝶临时维护导致失败，系统记录失败原因，次日凌晨自动重试，或财务上班后手动触发重新同步。

## 时效性声明

本章内容基于 **.NET 5.0**、**ABP 4.4.0**、**MailKit 2.15**、**Polly 7.2** 编写。IHttpClientFactory 在 .NET 各版本中保持稳定。在 .NET 6+ 中可使用 `builder.Services.AddHttpClient<T>()` 简化注册。阿里云 SDK 版本更新频繁，建议使用最新稳定版。

**银行银联说明**：§9 中的银行接口以合作银行提供的接口文档为准，不同银行的 API 路径和签名方案可能不同。

**微信小程序支付说明**：小程序端调起支付（`wx.requestPayment`）由银行/银联侧 SDK 决定，后端不负责商户密钥管理。

**金蝶选型说明**：§10 基于**金蝶云·星空（K3 Cloud）** WebAPI 编写，这是房地产/养老业财最常用的金蝶产品之一。支持 WebAPI 开放接口（登录鉴权、单据 Save/Submit/Audit、凭证生成）、应收款管理（AR）、收款单/退款单/转账单等财务单据、多组织多账簿（适配院区/楼栋多主体核算）。若实际部署为 EAS 或苍穹，对接模式类似但 API 路径不同，以金蝶实施方文档为准。
