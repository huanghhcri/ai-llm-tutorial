## 为什么需要学这个？

很多初学者拿到 ABP 后，所有实体一律继承 `AggregateRoot<Guid>`，所有查询直接 `.ToList()`，所有逻辑塞进应用服务。这三个"偷懒"行为会导致：

- **审计字段缺失**——无法追溯"谁在什么时候创建/修改了这条记录"
- **值对象语义丢失**——地址本应是不可变概念，却被当成可随意修改的字段集合
- **应用服务膨胀**——一个 `ElderAppService` 动辄 800 行，业务规则散落各处
- **管道绕过**——直接 `.ToList()` 不走 ABP 的全局过滤器（多租户、软删除），导致数据泄露

本章逐一解决这些问题。

## 1. 实体基类选型指南

### 1.1 ABP 实体基类家族

ABP 提供了一套分层的实体基类，审计能力逐级递增：

| 基类 | 审计字段 | 软删除 | 多租户 | 适用场景 |
|------|----------|--------|--------|----------|
| `Entity<TKey>` | 无 | 可选 | 可选 | 极简实体，如字典项 |
| `CreationAuditedEntity<TKey>` | `CreationTime` `CreatorId` | 可选 | 可选 | 只需记录创建信息 |
| `AuditedEntity<TKey>` | 创建 + `LastModificationTime` `LastModifierId` | 可选 | 可选 | 需要完整修改记录 |
| `FullAuditedEntity<TKey>` | 全部审计字段 | ✅ 内置 | 可选 | 核心业务实体 |
| `AggregateRoot<TKey>` | 无（可组合） | 可选 | 可选 | 聚合根，支持领域事件 |
| `CreationAuditedAggregateRoot<TKey>` | 创建审计 | 可选 | 可选 | 聚合根 + 创建审计 |
| `AuditedAggregateRoot<TKey>` | 创建 + 修改审计 | 可选 | 可选 | 聚合根 + 完整修改审计 |
| `FullAuditedAggregateRoot<TKey>` | 全部审计字段 | ✅ 内置 | 可选 | 核心聚合根 |

### 1.2 选型决策树

```
该实体是聚合根吗？
├── 是 → 需要删除审计吗？
│   ├── 是 → FullAuditedAggregateRoot<TKey>
│   └── 否 → 需要修改审计吗？
│       ├── 是 → AuditedAggregateRoot<TKey>
│       └── 否 → 只需创建审计？
│           ├── 是 → CreationAuditedAggregateRoot<TKey>
│           └── 否 → AggregateRoot<TKey>
└── 否 → 需要删除审计吗？
    ├── 是 → FullAuditedEntity<TKey>
    └── 否 → AuditedEntity<TKey> 或 CreationAuditedEntity<TKey>
```

### 1.3 养老院场景选型示例

```csharp
// 长者是核心聚合根，需要完整审计和软删除
public class Elder : FullAuditedAggregateRoot<Guid>, IMultiTenant
{
    public Guid? TenantId { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
    public string Gender { get; set; }
    public DateTime CheckInDate { get; set; }
    public ElderStatus Status { get; set; }
}

// 护理记录属于长者聚合，只需创建审计
public class CareRecord : CreationAuditedEntity<Guid>, IMultiTenant
{
    public Guid? TenantId { get; set; }
    public Guid ElderId { get; set; }
    public string Content { get; set; }
    public CareType Type { get; set; }
}

// 费用账单需要完整审计（财务追溯）
public class Bill : FullAuditedAggregateRoot<Guid>, IMultiTenant
{
    public Guid? TenantId { get; set; }
    public Guid ElderId { get; set; }
    public decimal Amount { get; set; }
    public BillStatus Status { get; set; }
    public BillingPeriod Period { get; set; } // 值对象
}
```

### 1.4 `IMultiTenant` 与 `ISoftDelete` 接口

`IMultiTenant` 接口只有一个属性 `Guid? TenantId`。当实体继承 `FullAuditedAggregateRoot` 时，软删除已内置实现 `ISoftDelete`（包含 `IsDeleted`、`DeleterId`、`DeletionTime`）。对于非 `FullAudited` 基类，如果需要软删除，需手动实现：

```csharp
// 手动实现软删除（适用于继承 AggregateRoot 的实体）
public class Room : AggregateRoot<Guid>, IMultiTenant, ISoftDelete
{
    public Guid? TenantId { get; set; }
    public bool IsDeleted { get; set; }          // ISoftDelete 要求
    public Guid? DeleterId { get; set; }          // ABP 自动填充
    public DateTime? DeletionTime { get; set; }   // ABP 自动填充
    public string RoomNumber { get; set; }
    public int Floor { get; set; }
    public RoomType Type { get; set; }
}
```

> **最佳实践**：养老院系统中，长者、房间、账单等核心实体一律使用 `FullAuditedAggregateRoot`；字典类（如餐食类型、活动类型）使用 `Entity<Guid>` 即可。

## 2. 值对象 vs 实体

### 2.1 核心区别

| 特征 | 实体（Entity） | 值对象（ValueObject） |
|------|---------------|----------------------|
| 标识 | 有唯一 ID | 无独立 ID |
| 相等性 | 比较 ID | 比较所有属性 |
| 生命周期 | 独立管理 | 随所属实体存在 |
| 可变性 | 可修改属性 | 不可变（创建后不改） |
| 示例 | Elder、Bill | Address、BillingPeriod |

### 2.2 ABP `ValueObject` 基类

```csharp
using Volo.Abp.Domain.Values;

// 地址——典型的值对象
public class Address : ValueObject
{
    public string Province { get; }
    public string City { get; }
    public string District { get; }
    public string Street { get; }
    public string ZipCode { get; }

    // 构造函数私有化，防止外部随意创建
    private Address() { }

    public Address(string province, string city, string district,
                   string street, string zipCode)
    {
        Province = Check.NotNullOrWhiteSpace(province, nameof(province));
        City = Check.NotNullOrWhiteSpace(city, nameof(city));
        District = Check.NotNullOrWhiteSpace(district, nameof(district));
        Street = street;
        ZipCode = zipCode;
    }

    // ABP 要求重写：定义值对象的"相等"含义
    protected override IEnumerable<object> GetAtomicValues()
    {
        yield return Province;
        yield return City;
        yield return District;
        yield return Street;
        yield return ZipCode;
    }
}
```

### 2.3 在实体中使用值对象

```csharp
public class Elder : FullAuditedAggregateRoot<Guid>, IMultiTenant
{
    public Guid? TenantId { get; set; }
    public string Name { get; set; }
    public Address HomeAddress { get; set; }       // 值对象
    public Address EmergencyAddress { get; set; }  // 同一值对象可复用
}
```

EF Core 默认将值对象映射为 Owned Entity Type，在 `DbContext` 中配置：

```csharp
builder.Entity<Elder>(b =>
{
    b.OwnsOne(e => e.HomeAddress, a =>
    {
        a.Property(x => x.Province).HasMaxLength(64).HasColumnName("HomeProvince");
        a.Property(x => x.City).HasMaxLength(64).HasColumnName("HomeCity");
        a.Property(x => x.District).HasMaxLength(64).HasColumnName("HomeDistrict");
        a.Property(x => x.Street).HasMaxLength(256).HasColumnName("HomeStreet");
        a.Property(x => x.ZipCode).HasMaxLength(16).HasColumnName("HomeZipCode");
    });
    b.OwnsOne(e => e.EmergencyAddress, a =>
    {
        a.Property(x => x.Province).HasMaxLength(64).HasColumnName("EmergencyProvince");
        a.Property(x => x.City).HasMaxLength(64).HasColumnName("EmergencyCity");
        a.Property(x => x.District).HasMaxLength(64).HasColumnName("EmergencyDistrict");
        a.Property(x => x.Street).HasMaxLength(256).HasColumnName("EmergencyStreet");
        a.Property(x => x.ZipCode).HasMaxLength(16).HasColumnName("EmergencyZipCode");
    });
});
```

> **注意**：值对象是**不可变**的。如果长者搬家，应该创建新的 `Address` 实例赋值给 `HomeAddress`，而不是修改原对象的属性。

## 3. 领域服务 vs 应用服务

### 3.1 边界划分

| 维度 | 领域服务（Domain Service） | 应用服务（App Service） |
|------|--------------------------|----------------------|
| 层次 | 领域层 | 应用层 |
| 职责 | 封装**不自然属于任何单个实体**的业务规则 | 编排用例，协调多个领域对象 |
| 依赖 | 不依赖仓储、不依赖 DTO | 依赖仓储、依赖 DTO 映射 |
| 示例 | 计费规则、入住条件校验 | 创建账单用例、查询长者列表 |
| 返回值 | 领域对象或 void | DTO 或 void |

### 3.2 领域服务示例：`BillManager`

```csharp
using Volo.Abp.Domain.Services;

public class BillManager : DomainService
{
    // 领域服务不注入仓储！只接收实体参数
    public Bill GenerateMonthlyBill(Elder elder, List<CareRecord> records,
                                     DateTime periodStart, DateTime periodEnd)
    {
        var baseFee = CalculateBaseFee(elder);
        var careFee = records.Sum(r => GetCareFee(r.Type));
        var totalAmount = baseFee + careFee;

        var billingPeriod = new BillingPeriod(periodStart, periodEnd);

        var bill = new Bill(
            GuidGenerator.Create(),
            elder.Id,
            totalAmount,
            billingPeriod
        );

        return bill;
    }

    private decimal CalculateBaseFee(Elder elder)
    {
        return elder.Type switch
        {
            ElderType.SelfCare => 3000m,
            ElderType.HalfCare => 5000m,
            ElderType.FullCare => 8000m,
            _ => throw new BusinessException("YL:InvalidElderType")
        };
    }

    private decimal GetCareFee(CareType type)
    {
        return type switch
        {
            CareType.Medication => 50m,
            CareType.PhysicalTherapy => 120m,
            CareType.Nursing => 80m,
            _ => 0m
        };
    }
}
```

### 3.3 应用服务示例：`BillAppService`

```csharp
public class BillAppService : YlsAppService, IBillAppService
{
    private readonly IRepository<Elder, Guid> _elderRepository;
    private readonly IRepository<Bill, Guid> _billRepository;
    private readonly IRepository<CareRecord, Guid> _careRecordRepository;
    private readonly BillManager _billManager;

    public BillAppService(
        IRepository<Elder, Guid> elderRepository,
        IRepository<Bill, Guid> billRepository,
        IRepository<CareRecord, Guid> careRecordRepository,
        BillManager billManager)
    {
        _elderRepository = elderRepository;
        _billRepository = billRepository;
        _careRecordRepository = careRecordRepository;
        _billManager = billManager;
    }

    public async Task<BillDto> GenerateBillAsync(GenerateBillInput input)
    {
        // 1. 应用服务负责"编排"——从仓储获取数据
        var elder = await _elderRepository.GetAsync(input.ElderId);

        var records = await AsyncExecuter.ToListAsync(
            (await _careRecordRepository.GetQueryableAsync())
                .Where(r => r.ElderId == input.ElderId
                    && r.CreationTime >= input.PeriodStart
                    && r.CreationTime <= input.PeriodEnd)
        );

        // 2. 调用领域服务执行业务规则
        var bill = _billManager.GenerateMonthlyBill(
            elder, records, input.PeriodStart, input.PeriodEnd);

        // 3. 持久化
        await _billRepository.InsertAsync(bill);

        // 4. 返回 DTO
        return ObjectMapper.Map<Bill, BillDto>(bill);
    }
}
```

> **关键原则**：应用服务做"搬运"（取数据、调领域服务、存数据、返 DTO），领域服务做"判断"（计费规则、业务约束）。不要让应用服务直接写 `if (amount > 1000)` 这类业务规则。

## 4. 仓储接口设计规范

### 4.1 ABP 内置仓储

ABP 为每个实体自动注册 `IRepository<TEntity, TKey>`，提供标准 CRUD。对于简单场景可以直接注入使用。

### 4.2 自定义仓储接口

当内置仓储无法满足查询需求时，定义专用接口：

```csharp
using Volo.Abp.Domain.Repositories;

// 继承 IRepository 扩展自定义方法
public interface IElderRepository : IRepository<Elder, Guid>
{
    Task<List<Elder>> GetListByStatusAsync(ElderStatus status);

    Task<Elder> FindByNameAsync(string name);

    Task<List<ElderWithBillDto>> GetEldersWithPendingBillsAsync();
}
```

### 4.3 接口设计原则

1. **返回实体，不返回 DTO**——仓储属于领域层，不依赖应用层概念
2. **方法名表达业务意图**——`GetListByStatusAsync` 优于 `QueryAsync`
3. **不暴露 IQueryable**——仓储接口不应返回 `IQueryable<T>`，否则调用方可能写出穿透聚合边界的查询
4. **参数尽量用值类型**——传 `ElderStatus` 而非传一串 `string`

## 5. AsyncExecuter 为什么必须用

### 5.1 问题复现

很多开发者习惯这样写查询：

```csharp
// ❌ 错误写法：直接调用 LINQ 的 ToList()
var elders = (await _elderRepository.GetQueryableAsync())
    .Where(e => e.Status == ElderStatus.Active)
    .ToList();  // 绕过了 ABP 管道！
```

这种写法**绕过了 ABP 的 `IAsyncQueryableExecuter`**，后果包括：

- **多租户过滤器失效**——可能查到其他租户的数据
- **软删除过滤器失效**——已删除的记录也会返回
- **审计拦截器不触发**——操作不会记录到审计日志

### 5.2 正确写法

```csharp
// ✅ 正确：使用 AsyncExecuter
var elders = await AsyncExecuter.ToListAsync(
    (await _elderRepository.GetQueryableAsync())
        .Where(e => e.Status == ElderStatus.Active)
);
```

### 5.3 `IAsyncQueryableExecuter` 原理

ABP 的 `IAsyncQueryableExecuter` 是一个拦截器，它在执行查询前会：

1. 应用全局查询过滤器（`IDataFilter`）
2. 将 LINQ 表达式树转换为正确的 SQL
3. 调用底层 EF Core 的异步执行方法

`ApplicationService` 基类内置了 `AsyncExecuter` 属性，直接使用即可。如果在非应用服务类中使用，需注入 `IAsyncQueryableExecuter`：

```csharp
public class BillManager : DomainService
{
    private readonly IAsyncQueryableExecuter _asyncExecuter;

    public BillManager(IAsyncQueryableExecuter asyncExecuter)
    {
        _asyncExecuter = asyncExecuter;
    }

    public async Task<List<Bill>> GetPendingBillsAsync(IRepository<Bill, Guid> billRepo)
    {
        var queryable = (await billRepo.GetQueryableAsync())
            .Where(b => b.Status == BillStatus.Pending);

        return await _asyncExecuter.ToListAsync(queryable);
    }
}
```

### 5.4 常用异步扩展方法

```csharp
// 列表
await AsyncExecuter.ToListAsync(queryable);

// 单个
await AsyncExecuter.FirstOrDefaultAsync(queryable.Where(x => x.Id == id));

// 计数
await AsyncExecuter.CountAsync(queryable);

// 是否存在
await AsyncExecuter.AnyAsync(queryable.Where(x => x.Status == status));
```

## 6. 自定义仓储基类 `EfCoreYlsRepositoryBase`

### 6.1 设计动机

养老院系统中，多个仓储有共同的查询逻辑（如按租户过滤、按状态筛选）。将这些逻辑提取到基类，避免重复代码。

### 6.2 基类实现

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Linq.Expressions;
using System.Threading.Tasks;
using Microsoft.EntityFrameworkCore;
using Volo.Abp.Domain.Entities;
using Volo.Abp.Domain.Repositories.EntityFrameworkCore;
using Volo.Abp.EntityFrameworkCore;

/// <summary>
/// 养老院系统自定义仓储基类
/// 封装通用查询逻辑，各模块仓储继承此类
/// </summary>
public abstract class EfCoreYlsRepositoryBase<TDbContext, TEntity, TKey>
    : EfCoreRepository<TDbContext, TEntity, TKey>
    where TDbContext : IEfCoreDbContext
    where TEntity : class, IEntity<TKey>
{
    protected EfCoreYlsRepositoryBase(
        IDbContextProvider<TDbContext> dbContextProvider)
        : base(dbContextProvider)
    {
    }

    /// <summary>
    /// 按条件分页查询，支持排序
    /// </summary>
    protected async Task<(List<TEntity> Items, int TotalCount)> GetPagedListAsync(
        Expression<Func<TEntity, bool>> predicate,
        int skipCount,
        int maxResultCount,
        string sorting = null,
        bool includeDetails = false)
    {
        var queryable = includeDetails
            ? (await GetQueryableAsync()).IncludeDetails()
            : await GetQueryableAsync();

        queryable = queryable.Where(predicate);

        if (!string.IsNullOrWhiteSpace(sorting))
        {
            queryable = queryable.OrderBy(sorting);
        }

        var totalCount = await AsyncExecuter.CountAsync(queryable);
        var items = await AsyncExecuter.ToListAsync(
            queryable.PageBy(skipCount, maxResultCount));

        return (items, totalCount);
    }

    /// <summary>
    /// 获取带完整关联数据的实体
    /// </summary>
    protected virtual async Task<TEntity> GetWithDetailsAsync(
        TKey id,
        Expression<Func<TEntity, bool>> additionalPredicate = null)
    {
        var queryable = (await GetQueryableAsync()).IncludeDetails();

        if (additionalPredicate != null)
        {
            queryable = queryable.Where(additionalPredicate);
        }

        return await AsyncExecuter.FirstOrDefaultAsync(
            queryable.Where(e => EqualityComparer<TKey>.Default.Equals(e.Id, id)));
    }
}
```

### 6.3 子仓储继承

```csharp
// 具体仓储继承自定义基类
public class EfCoreElderRepository
    : EfCoreYlsRepositoryBase<YlsDbContext, Elder, Guid>, IElderRepository
{
    public EfCoreElderRepository(IDbContextProvider<YlsDbContext> dbContextProvider)
        : base(dbContextProvider)
    {
    }

    public async Task<List<Elder>> GetListByStatusAsync(ElderStatus status)
    {
        return await AsyncExecuter.ToListAsync(
            (await GetQueryableAsync())
                .Where(e => e.Status == status)
                .OrderBy(e => e.Name));
    }

    public async Task<Elder> FindByNameAsync(string name)
    {
        return await AsyncExecuter.FirstOrDefaultAsync(
            (await GetQueryableAsync())
                .Where(e => e.Name == name));
    }

    public async Task<List<ElderWithBillDto>> GetEldersWithPendingBillsAsync()
    {
        var dbContext = await GetDbContextAsync();
        var queryable = from elder in dbContext.Set<Elder>()
                        join bill in dbContext.Set<Bill>()
                            on elder.Id equals bill.ElderId
                        where bill.Status == BillStatus.Pending
                        select new ElderWithBillDto
                        {
                            ElderId = elder.Id,
                            ElderName = elder.Name,
                            BillAmount = bill.Amount
                        };
        // 必须使用 AsyncExecuter，确保全局过滤器（租户/软删除）生效
        return await AsyncExecuter.ToListAsync(queryable);
    }
}
```

## 7. 原始 SQL 查询

### 7.1 何时使用

- **复杂报表统计**——多表联查、窗口函数、CTE 等 LINQ 难以表达的查询
- **存储过程调用**——调用数据库中已有的存储过程
- **性能优化**——EF Core 生成的 SQL 不够高效时
- **数据库特有功能**——全文检索、地理空间查询等

> **警告**：原始 SQL 绕过了 EF Core 的变更追踪和 ABP 的全局过滤器。使用时必须手动处理多租户和软删除。

### 7.2 `FromSqlRaw`——查询

```csharp
public class EfCoreReportRepository
    : EfCoreRepository<YlsDbContext, Report, Guid>, IReportRepository
{
    public EfCoreReportRepository(IDbContextProvider<YlsDbContext> dbContextProvider)
        : base(dbContextProvider) { }

    public async Task<List<MonthlyBillSummary>> GetMonthlyBillSummaryAsync(
        int year, int month)
    {
        var dbContext = await GetDbContextAsync();

        var sql = @"
            SELECT
                e.Name AS ElderName,
                b.Amount,
                b.Status,
                b.PeriodStart,
                b.PeriodEnd
            FROM Bills b
            INNER JOIN Elders e ON b.ElderId = e.Id
            WHERE YEAR(b.PeriodStart) = {0}
              AND MONTH(b.PeriodStart) = {1}
              AND e.IsDeleted = 0           -- 手动过滤软删除
              AND e.TenantId = {2}          -- 手动过滤租户
            ORDER BY e.Name";

        var tenantId = CurrentTenant.Id;

        return await dbContext.Set<MonthlyBillSummary>()
            .FromSqlRaw(sql, year, month, tenantId)
            .ToListAsync();
    }
}
```

### 7.3 `ExecuteSqlRawAsync`——写入

```csharp
public async Task<int> BatchUpdateBillStatusAsync(List<Guid> billIds, BillStatus newStatus)
{
    var dbContext = await GetDbContextAsync();
    var idList = string.Join(",", billIds.Select(id => $"'{id}'"));
    var statusValue = (int)newStatus;

    var sql = $"UPDATE Bills SET Status = {statusValue} WHERE Id IN ({idList})";

    return await dbContext.Database.ExecuteSqlRawAsync(sql);
}
```

### 7.4 参数化防注入

```csharp
// ✅ 正确：使用参数化查询
var sql = "SELECT * FROM Elders WHERE Name = {0} AND Age > {1}";
var result = await dbContext.Set<Elder>()
    .FromSqlRaw(sql, inputName, inputAge)
    .ToListAsync();
```

## 8. 实战：设计长者健康评估模块

### 8.1 需求分析

养老院需要定期对长者进行健康评估，每次评估包含多个评估项（如血压、血糖、认知能力等）。设计要点：

- `HealthAssessment`（健康评估）是聚合根
- `AssessmentItem`（评估项）是聚合内的实体
- 每次评估关联一个长者
- 评估项需要支持自定义扩展

### 8.2 实体定义

```csharp
using System;
using System.Collections.Generic;
using System.Collections.ObjectModel;
using Volo.Abp;
using Volo.Abp.Domain.Entities.Auditing;
using Volo.Abp.MultiTenancy;

/// <summary>
/// 健康评估——聚合根
/// </summary>
public class HealthAssessment : FullAuditedAggregateRoot<Guid>, IMultiTenant
{
    public Guid? TenantId { get; set; }

    /// <summary>关联长者</summary>
    public Guid ElderId { get; set; }

    /// <summary>评估日期</summary>
    public DateTime AssessmentDate { get; set; }

    /// <summary>评估等级</summary>
    public HealthLevel Level { get; set; }

    /// <summary>评估医生</summary>
    public string AssessorName { get; set; }

    /// <summary>综合评语</summary>
    public string Summary { get; set; }

    /// <summary>评估项集合</summary>
    public ICollection<AssessmentItem> Items { get; set; }

    private HealthAssessment() { }

    public HealthAssessment(
        Guid id,
        Guid elderId,
        DateTime assessmentDate,
        string assessorName)
        : base(id)
    {
        ElderId = elderId;
        AssessmentDate = assessmentDate;
        AssessorName = Check.NotNullOrWhiteSpace(assessorName, nameof(assessorName));
        Items = new Collection<AssessmentItem>();
    }

    /// <summary>添加评估项（聚合内部操作）</summary>
    public AssessmentItem AddItem(string name, decimal value, string unit, string remark = null)
    {
        var item = new AssessmentItem(GuidGenerator.Create(), Id, name, value, unit, remark);
        Items.Add(item);
        return item;
    }

    /// <summary>根据评估项总分计算健康等级</summary>
    public void EvaluateLevel()
    {
        var totalScore = Items.Sum(i => i.Value);
        Level = totalScore switch
        {
            >= 90 => HealthLevel.Excellent,
            >= 70 => HealthLevel.Good,
            >= 50 => HealthLevel.Fair,
            _ => HealthLevel.Poor
        };
    }
}

/// <summary>
/// 评估项——聚合内实体（非独立聚合根）
/// </summary>
public class AssessmentItem : CreationAuditedEntity<Guid>, IMultiTenant

    public Guid AssessmentId { get; set; }

    /// <summary>指标数值</summary>
    public decimal Value { get; set; }

    /// <summary>单位</summary>
    public string Unit { get; set; }

    /// <summary>备注</summary>
    public string Remark { get; set; }

    private AssessmentItem() { }

    public AssessmentItem(
        Guid id,
        Guid assessmentId,
        string name,
        decimal value,
        string unit,
        string remark = null)
        : base(id)
    {
        AssessmentId = assessmentId;
        Name = Check.NotNullOrWhiteSpace(name, nameof(name));
        Value = value;
        Unit = Check.NotNullOrWhiteSpace(unit, nameof(unit));
        Remark = remark;
    }
}

/// <summary>
/// 健康等级枚举
/// </summary>
public enum HealthLevel
{
    Poor = 0,
    Fair = 1,
    Good = 2,
    Excellent = 3
}
```

### 8.4 仓储接口与实现

```csharp
// 接口（领域层）
public interface IHealthAssessmentRepository
    : IRepository<HealthAssessment, Guid>
{
    Task<HealthAssessment> GetWithItemsAsync(Guid id);

    Task<List<HealthAssessment>> GetListByElderAsync(
        Guid elderId, int maxResultCount = 10);

    Task<List<HealthAssessment>> GetRecentAssessmentsAsync(
        Guid elderId, DateTime since);
}

// 实现（EntityFrameworkCore 层）
public class EfCoreHealthAssessmentRepository
    : EfCoreYlsRepositoryBase<YlsDbContext, HealthAssessment, Guid>,
      IHealthAssessmentRepository
{
    public EfCoreHealthAssessmentRepository(
        IDbContextProvider<YlsDbContext> dbContextProvider)
        : base(dbContextProvider)
    {
    }

    public async Task<HealthAssessment> GetWithItemsAsync(Guid id)
    {
        var queryable = (await GetQueryableAsync())
            .Include(a => a.Items)
            .Where(a => a.Id == id);
        // 使用 AsyncExecuter 确保全局过滤器生效
        return await AsyncExecuter.FirstOrDefaultAsync(queryable);
    }

    public async Task<List<HealthAssessment>> GetListByElderAsync(
        Guid elderId, int maxResultCount = 10)
    {
        return await AsyncExecuter.ToListAsync(
            (await GetQueryableAsync())
                .Include(a => a.Items)
                .Where(a => a.ElderId == elderId)
                .OrderByDescending(a => a.AssessmentDate)
                .Take(maxResultCount));
    }

    public async Task<List<HealthAssessment>> GetRecentAssessmentsAsync(
        Guid elderId, DateTime since)
    {
        return await AsyncExecuter.ToListAsync(
            (await GetQueryableAsync())
                .Where(a => a.ElderId == elderId && a.AssessmentDate >= since)
                .OrderByDescending(a => a.AssessmentDate));
    }
}
```

### 8.5 应用服务

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using Volo.Abp.Application.Services;

public interface IHealthAssessmentAppService
    : IApplicationService
{
    Task<HealthAssessmentDto> CreateAsync(CreateHealthAssessmentInput input);
    Task<HealthAssessmentDto> GetAsync(Guid id);
    Task<List<HealthAssessmentDto>> GetListByElderAsync(Guid elderId);
}

public class HealthAssessmentAppService
    : YlsAppService, IHealthAssessmentAppService
{
    private readonly IHealthAssessmentRepository _assessmentRepository;
    private readonly IRepository<Elder, Guid> _elderRepository;

    public HealthAssessmentAppService(
        IHealthAssessmentRepository assessmentRepository,
        IRepository<Elder, Guid> elderRepository)
    {
        _assessmentRepository = assessmentRepository;
        _elderRepository = elderRepository;
    }

    public async Task<HealthAssessmentDto> CreateAsync(CreateHealthAssessmentInput input)
    {
        // 验证长者存在
        var elder = await _elderRepository.GetAsync(input.ElderId);

        // 创建评估聚合
        var assessment = new HealthAssessment(
            GuidGenerator.Create(),
            input.ElderId,
            input.AssessmentDate,
            input.AssessorName
        );

        // 添加评估项
        foreach (var itemInput in input.Items)
        {
            assessment.AddItem(
                itemInput.Name,
                itemInput.Value,
                itemInput.Unit,
                itemInput.Remark
            );
        }

        // 计算等级
        assessment.EvaluateLevel();
        assessment.Summary = input.Summary;

        // 持久化（EF Core 自动级联保存 Items）
        await _assessmentRepository.InsertAsync(assessment, autoSave: true);

        return ObjectMapper.Map<HealthAssessment, HealthAssessmentDto>(assessment);
    }

    public async Task<HealthAssessmentDto> GetAsync(Guid id)
    {
        var assessment = await _assessmentRepository.GetWithItemsAsync(id);
        return ObjectMapper.Map<HealthAssessment, HealthAssessmentDto>(assessment);
    }

    public async Task<List<HealthAssessmentDto>> GetListByElderAsync(Guid elderId)
    {
        var list = await _assessmentRepository.GetListByElderAsync(elderId);
        return ObjectMapper.Map<List<HealthAssessment>, List<HealthAssessmentDto>>(list);
    }
}
```

## 常见错误与最佳实践

| # | 错误写法 | 正确写法 | 为什么错 |
|---|---------|---------|---------|
| 1 | 直接 `.ToList()` 绕过过滤器 | 使用 `AsyncExecuter.ToListAsync()` | 软删除和租户过滤器失效，可能泄露已删除/其他租户数据 |
| 2 | 所有实体继承 `FullAuditedAggregateRoot` | 根据选型决策树选择基类，字典类用 `Entity<Guid>` | Full 版增加 6 个审计字段，不需要审计的表浪费存储 |
| 3 | 值对象未重写 `GetAtomicValues()` | 确保所有属性都 yield return | 值对象比较始终为 false，影响业务逻辑判断 |
| 4 | 业务规则写在应用服务中（超 500 行） | 提取为领域服务 | 应用服务膨胀、无法复用、难以单元测试 |
| 5 | 原始 SQL 未手动过滤 TenantId | SQL 中加 `AND TenantId = @tenantId` | 原始 SQL 绕过全局过滤器，泄露其他租户数据 |
| 6 | 仓储接口返回 `IQueryable` | 返回实体列表，内部封装查询 | 调用方绕过聚合边界构造任意查询，破坏封装性 |
| 7 | 聚合内实体直接操作仓储 | 通过聚合根方法操作内部实体 | 破坏聚合一致性边界，业务规则可能被绕过 |
| 8 | 领域服务直接注入 `IRepository` | 通过方法参数接收查询结果 | 领域层依赖基础设施层，违反 DDD 依赖方向 |

## 本章小结

本章围绕 ABP 实体设计与仓储模式的进阶主题展开：

1. **实体基类选型**不是小事——选错基类会导致审计缺失、软删除无效等严重问题
2. **值对象**让领域模型更精确——地址、金额区间等不可变概念应建模为值对象
3. **领域服务与应用服务有明确边界**——业务规则归属领域服务，用例编排归属应用服务
4. **仓储接口设计**要遵循 DDD 规范——不暴露 IQueryable，方法名体现业务意图
5. **`AsyncExecuter` 是必选项**——绕过它等于绕过 ABP 的全部安全管道
6. **自定义仓储基类**能大幅减少重复代码
7. **原始 SQL** 是最后手段——使用时必须手动处理过滤器

## 面试题

### 面试题 1（初级 / 概念题）
**题目**：FullAuditedAggregateRoot 和 AuditedAggregateRoot 有什么区别？

**参考答案**：`FullAuditedAggregateRoot` 在 `AuditedAggregateRoot` 基础上额外实现了 `ISoftDelete` 接口，包含 `IsDeleted`、`DeleterId`、`DeletionTime` 三个字段，支持逻辑删除而非物理删除。养老院场景：长者实体用 `FullAuditedAggregateRoot`，退住时只标记删除，历史数据仍可查询；而餐食类型等字典表用 `Entity<Guid>` 即可，不需要审计也不需要软删除。

### 面试题 2（初级 / 概念题）
**题目**：值对象和实体有什么区别？

**参考答案**：实体有唯一标识（Id），生命周期内属性可变，通过 ID 判断相等；值对象无唯一标识，不可变，通过属性值判断相等。养老院场景：`Elder`（长者）是实体，有唯一 Guid，姓名可以修改；`Address`（地址）是值对象，换地址就是新的值对象，ABP 中通过 `ValueObject.GetAtomicValues()` 判断相等性。

### 面试题 3（中级 / 概念题）
**题目**：领域服务和应用服务的边界在哪里？

**参考答案**：应用服务负责用例编排（取数据、调服务、返 DTO、权限检查），是"流程指挥官"；领域服务包含不归属于单个实体的核心业务规则，是"业务专家"。养老院场景：`BillAppService` 编排入住流程（调用领域服务计算账单 → 保存 → 通知），`BillManager` 领域服务负责"根据护理等级和天数计算账单金额"的纯业务规则。领域服务不应依赖仓储，应用服务不应包含计算逻辑。

### 面试题 4（中级 / 概念题）
**题目**：为什么仓储接口不应返回 IQueryable？

**参考答案**：返回 `IQueryable` 会让调用方绕过聚合边界，直接构造任意查询，破坏封装性。且不同 ORM 的 `IQueryable` 实现不同，不利于切换底层实现。正确做法是返回实体列表或 DTO，内部封装查询条件。养老院场景：`IElderRepository.GetByNursingLevelAsync(level)` 封装查询逻辑，调用方不需要知道底层是 EF Core 还是 Dapper。

### 面试题 5（中级 / 概念题）
**题目**：AsyncExecuter 的作用是什么？为什么不能直接调用 .ToList()？

**参考答案**：`IAsyncQueryableExecuter` 是 ABP 的异步查询执行器，在执行前注入全局过滤器（多租户、软删除等）。直接调用 `.ToList()` 或 `.FirstOrDefault()` 会绕过 ABP 的执行管道，导致过滤器失效。养老院场景：不用 AsyncExecuter 查询长者列表，可能返回已退住的长者（软删除失效）或其他养老院的长者（租户过滤失效），造成数据泄露。

### 面试题 6（初级 / 概念题）
**题目**：值对象在 EF Core 中如何映射？

**参考答案**：使用 Owned Entity Type，在 `DbContext` 的 `OnModelCreating` 中通过 `OwnsOne` 配置映射。值对象的属性会映射到所属实体的表中，以列前缀区分。养老院场景：`Elder` 实体拥有 `Address` 值对象，映射后数据库表中会有 `Address_City`、`Address_Street` 等列，而不是单独的地址表。

### 面试题 7（初级 / 概念题）
**题目**：FromSqlRaw 和 FromSqlInterpolated 有什么区别？

**参考答案**：`FromSqlRaw` 接受原始 SQL 字符串和参数数组，参数化防注入；`FromSqlInterpolated` 接受内插字符串，EF Core 自动将插值参数化。两者都安全，但 `FromSqlRaw` 在动态构造 SQL 时更灵活。养老院场景：复杂统计报表（如按月统计各护理等级的收入）可能需要 `FromSqlRaw` 调用存储过程。

### 面试题 8（高级 / 场景题）
**题目**：什么时候应该使用原始 SQL 查询？

**参考答案**：复杂窗口函数、CTE 递归查询、存储过程调用、数据库特有功能（全文检索）、EF Core 生成的 SQL 性能不佳时。使用时需手动处理多租户和软删除过滤。养老院场景：生成"连续 3 个月未缴费长者"报表需要 LAG 窗口函数，EF Core 5.0 不支持，必须用原始 SQL。注意：原始 SQL 会绕过全局过滤器，需手动加 `AND TenantId = @tenantId AND IsDeleted = 0`。

### 面试题 9（初级 / 概念题）
**题目**：聚合根的职责是什么？外部能直接操作聚合内实体吗？

**参考答案**：聚合根是一致性边界的核心，外部只能通过聚合根访问内部实体。内部实体的增删改查必须通过聚合根的方法暴露。养老院场景：`HealthAssessment`（健康评估）是聚合根，`AssessmentItem`（评估项）是聚合内实体，添加评估项必须通过 `assessment.AddItem()` 方法，保证"每个评估至少包含一项"的业务规则不被绕过。

### 面试题 10（中级 / 概念题）
**题目**：IMultiTenant 接口的作用是什么？

**参考答案**：`IMultiTenant` 让 ABP 自动为实体添加租户过滤，查询时只返回当前租户的数据。底层通过 EF Core 全局查询过滤器 `WHERE TenantId = @currentTenantId` 实现。养老院系统中所有业务实体都应实现此接口，因为每个养老院是独立租户。如果不实现，查询会返回所有养老院的数据，造成严重数据泄露。

### 面试题 11（中级 / 概念题）
**题目**：EfCoreYlsRepositoryBase 自定义仓储基类有什么作用？

**参考答案**：继承 `EfCoreRepository`，封装公共查询逻辑（分页、排序、软删除处理），子仓储按需调用。养老院场景：所有仓储都需要"按创建时间倒序 + 分页"的通用查询，封装在基类的 `GetPagedListAsync` 方法中，子仓储只需传入过滤条件。避免每个仓储重复写分页逻辑。

### 面试题 12（高级 / 场景题）
**题目**：领域服务能直接注入 IRepository 吗？

**参考答案**：严格的 DDD 规范中，领域服务不应依赖仓储，应通过方法参数接收查询结果。但 ABP 中为了实用，允许在领域服务中注入仓储。关键原则是领域服务只处理传入的实体数据，不负责数据获取的编排。养老院场景：`BillManager.CalculateFee(elder, days)` 接收已查出的 elder 实体，而不是在内部调用 `_elderRepo.GetAsync(id)`。

### 面试题 13（初级 / 概念题）
**题目**：CreationAuditedAggregateRoot 适用于什么场景？

**参考答案**：适用于创建后不再修改的实体，只记录创建人和创建时间，不追踪修改历史。养老院场景：护理记录、打卡日志、访客登记等，这些记录一旦创建即为定稿，不需要追踪谁修改过。使用 `CreationAuditedEntity` 而非 `FullAuditedEntity` 可以减少 3 个审计字段（ModifierId、LastModificationTime、DeleterId 等），节省存储。
