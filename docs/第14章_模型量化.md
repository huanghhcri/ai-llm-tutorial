### 2.1 量化的基本原理

#### 类比

养老院的健康评估表，原本用 0.00-100.00 的精度记录分数（需要 5 位数字）。实际上，"85.37 分"和"85 分"在护理评估中没有区别——用整数就够了（精度从 10000 档降到 100 档，信息损失极小）。

量化的本质：**用更少的比特位表示模型权重，牺牲微小精度换取巨大的存储和计算节省。**

#### 数值精度对比


#### 量化公式

```python
# 对称量化（Symmetric Quantization）
# 将浮点值映射到整数范围 [-127, 127]（INT8）或 [-7, 7]（INT4）
#
# 量化：x_int = round(x / scale)
# 反量化：x_float = x_int * scale
# 其中 scale = max(|x|) / (2^(bits-1) - 1)
#
# 类比：
# 原始分数 0-100 分 → 映射到 0-10 的等级
# 量化：85 分 → 8.5 → 四舍五入 → 9
# 反量化：9 → 90 分（近似值，误差 5 分）

# 示例
def quantize_symmetric(values: np.ndarray, bits: int = 4) -> tuple:
    """对称量化"""
    qmax = 2 ** (bits - 1) - 1   # INT4: 7, INT8: 127
    scale = np.max(np.abs(values)) / qmax
    quantized = np.round(values / scale).astype(np.int8)
    return quantized, scale

def dequantize(quantized: np.ndarray, scale: float) -> np.ndarray:
    """反量化"""
    return quantized.astype(np.float32) * scale

# 示例：量化一组权重
original_weights = np.array([0.5, -0.3, 0.8, -0.1, 0.6, -0.9, 0.2, -0.7])

# INT4 量化
q4, scale4 = quantize_symmetric(original_weights, bits=4)
recovered4 = dequantize(q4, scale4)

# INT8 量化
q8, scale8 = quantize_symmetric(original_weights, bits=8)
recovered8 = dequantize(q8, scale8)

print(f"原始值:     {original_weights}")
print(f"INT4 量化:  {q4}  (scale={scale4:.4f})")
print(f"INT4 还原:  {recovered4}")
print(f"INT4 误差:  {np.abs(original_weights - recovered4).round(4)}")
print(f"INT8 量化:  {q8}  (scale={scale8:.4f})")
print(f"INT8 还原:  {recovered8}")
print(f"INT8 误差:  {np.abs(original_weights - recovered8).round(4)}")
```

### 2.2 量化方法分类

#### 训练后量化（PTQ）vs 量化感知训练（QAT）

```python
# 训练后量化（Post-Training Quantization, PTQ）
# 模型训练完成后，直接对权重做量化
# 优点：不需要重新训练，速度快
# 缺点：精度损失比 QAT 大
# 代表：GPTQ、AWQ、GGUF
#
# 类比：
# 护理员培训完后，把她的知识"压缩"成简版手册
# 不需要重新培训，直接压缩

# 量化感知训练（Quantization-Aware Training, QAT）
# 在训练过程中模拟量化效果，让模型适应低精度
# 优点：精度损失最小
# 缺点：需要训练数据和计算资源
# 代表：bitsandbytes（训练时量化）
#
# 类比：
# 培训时就用简版教材，让护理员从一开始就适应
```

## 四、知识点总结

| 概念 | 一句话解释 | 关键要点 |
|------|-----------|---------|
| **量化** | 用更少比特表示模型权重 | FP16→INT4 节省 75% 显存，精度损失 <3% |
| **PTQ** | 训练后量化 | 不需要重新训练，直接对权重量化 |
| **QAT** | 量化感知训练 | 训练时模拟量化，精度最高 |
| **GPTQ** | 基于 Hessian 的 PTQ | GPU 推理，用校准数据优化量化参数 |
| **AWQ** | 激活感知量化 | 根据激活值重要性分配精度，比 GPTQ 更准 |
| **GGUF** | llama.cpp 模型格式 | CPU/GPU 混合推理，本地部署首选 |
| **bitsandbytes** | 动态量化库 | 一行代码量化，支持训练（QLoRA）和推理 |
| **Q4_K_M** | GGUF 4-bit 量化级别 | 精度和大小的最佳平衡点 |
| **校准数据** | 用于确定量化参数的数据 | 128-256 条代表性文本，不需要标签 |
| **反量化** | 将量化值还原为浮点 | 推理时自动执行，精度有损 |
| **分组量化** | 每组权重独立量化 | group_size=128 是常用值 |

## 五、本章面试题

### 题目 1：量化的原理是什么？为什么 INT4 量化还能保持不错的效果？

**难度**：⭐⭐  
**类型**：核心原理

**参考答案**：

量化的原理是用更少的比特位表示浮点权重值。以 INT4 为例：将 FP16 的 65536 个可能值映射到 INT4 的 16 个可能值（-8 到 7），每个权重从 2 字节压缩到 0.5 字节。INT4 还能保持效果的原因：① **权重分布冗余**——研究发现大模型的权重分布高度集中在 0 附近（近似正态分布），16 个离散值足以近似表示；② **信息冗余**——大模型有数十亿参数，单个参数的微小误差被大量参数的"多数投票"所抵消；③ **分组量化**——每 128 个权重共享一个缩放因子，降低了量化误差；④ **AWQ 的激活感知**——对重要权重保留更高精度。实测：7B 模型 INT4 量化后，在大多数基准上精度下降 <3%，对实际使用几乎无感知。

### 题目 2：GPTQ 和 AWQ 有什么区别？如何选择？

**难度**：⭐⭐  
**类型**：技术对比

**参考答案**：

GPTQ 和 AWQ 都是训练后量化（PTQ）方法，核心区别在于**量化策略**：① **GPTQ** 基于 Hessian 矩阵（二阶导数）来最小化量化误差——逐列量化，每量化一列后用 Hessian 信息补偿后续列的误差。优点是成熟稳定，缺点是对所有权重一视同仁。② **AWQ** 引入了"激活感知"——分析激活值的分布，对激活值大的权重通道（重要通道）保留更高精度，对不重要的通道做更激进量化。优点是同为 INT4 时精度比 GPTQ 高 0.5-1%，缺点是量化速度稍慢。选择建议：① 追求极致精度 → AWQ；② 追求量化速度 → GPTQ；③ vLLM 部署两者都支持。养老院场景中推荐 AWQ——精度更高，且 vLLM 对 AWQ 的支持很好。

---

### 题目 3：GGUF 格式的 Q4_K_M 是什么意思？不同量化级别如何选择？

**难度**：⭐  
**类型**：格式理解

**参考答案**：

GGUF 的量化级别命名规则：Q 表示量化（Quantized），数字表示平均比特数，K 表示 K-quant（分组量化），后缀表示精度级别：S=Small（最激进），M=Medium（平衡），L=Large（最保守）。Q4_K_M 表示"平均 4-bit 的中等精度 K-quant"。选择建议：① **追求最高精度** → Q8_0（几乎无损，但大小翻倍）；② **日常使用推荐** → Q4_K_M（精度和大小的最佳平衡，7B 模型仅 3.8GB）；③ **极致压缩** → Q3_K_M 或 Q2_K（大小最小但精度下降明显）；④ **GPU 推理** → 不需要 GGUF，直接用 GPTQ/AWQ。养老院推荐 Q4_K_M——3.8GB 大小可以在任何服务器上运行，精度损失约 3%，对护理评估任务几乎无影响。

### 题目 4：bitsandbytes 的 NF4 是什么？为什么比普通 INT4 好？

**难度**：⭐⭐  
**类型**：技术细节

**参考答案**：

NF4（NormalFloat 4-bit）是 QLoRA 论文提出的一种 4-bit 量化格式。与普通 INT4 的区别：INT4 的 16 个量化点均匀分布（-8, -7, ..., 7），而 NF4 的 16 个量化点按正态分布排列——中间密集、两端稀疏。为什么更好：神经网络的权重分布近似正态（大部分值在 0 附近，极端值很少），NF4 的量化点分布与之匹配，每个量化区间内的权重数量大致相等，量化误差更均匀、更小。效果：在相同 4-bit 精度下，NF4 的量化误差比 INT4 低约 0.5-1%。bitsandbytes 还支持"双重量化"（Double Quantization）——对量化常数本身再做一次量化，进一步压缩 0.5GB 左右。

### 题目 5：如何评估量化对模型质量的影响？

**难度**：⭐⭐  
**类型**：质量评估

**参考答案**：

评估方法：① **基准测试**——用 lm-eval-harness 在标准基准（MMLU、GSM8K 等）上比较量化前后的分数；② **领域测试**——在养老院的护理评估测试集上比较量化前后的输出质量（格式一致性、内容准确性）；③ **困惑度（Perplexity）**——衡量模型对文本的预测能力，困惑度越低越好。量化后困惑度升高越少说明精度保持越好；④ **人工评估**——让护理主管对量化前后的评估报告做盲评打分；⑤ **速度测试**——测量 tokens/s，确认量化后的推理速度提升。养老院建议：在自己的测试集上评估最重要——标准基准可能显示量化后分数下降 2%，但在护理评估任务上可能几乎无差异。

### 题目 7：量化后的模型还能继续微调吗？

**难度**：⭐  
**类型**：实践问题

**参考答案**：

取决于量化方式：① **bitsandbytes 量化** → 可以继续微调。QLoRA 就是在 4-bit 量化模型上加 LoRA 微调，这是 bitsandbytes 的核心用途；② **GPTQ/AWQ 量化** → 通常不直接微调，但可以在量化模型上加 LoRA（需要用兼容的库）；③ **GGUF 量化** → 不支持微调，只能用于推理。推荐工作流：先用 bitsandbytes（QLoRA）微调 → 保存 LoRA 适配器 → 合并到基座模型 → 用 GPTQ/AWQ/GGUF 量化合并后的模型 → 部署推理。这样既享受了微调的领域适配，又享受了量化的小体积和快速度。

### 题目 8：什么是 Group Size？它如何影响量化质量？

**难度**：⭐⭐  
**类型**：参数理解

**参考答案**：

Group Size（分组大小）是指多少个权重共享一个缩放因子（scale）和零点（zero_point）。默认值通常为 128。影响：① **group_size=128**——128 个权重共享一组量化参数，精度和大小的平衡点；② **group_size=64**——更小的分组，每个分组的量化更精确，但量化参数本身占的存储增加（总大小增大 5-10%）；③ **group_size=-1（无分组）**——整个权重矩阵共享一组量化参数，精度最差但大小最小。选择建议：① 追求精度 → group_size=64 或 32；② 追求平衡 → group_size=128（推荐）；③ 追求极致压缩 → group_size=-1（精度损失较大）。养老院推荐 group_size=128——标准配置，精度和大小都不极端。
